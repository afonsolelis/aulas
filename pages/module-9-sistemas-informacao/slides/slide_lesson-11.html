SRP — Single Responsibility Principle

❌ Violação

public class Pedido {
public void Calcular() { ... }
public void SalvarNoBanco() { ... }
public void EnviarEmail() { ... }
public void GerarPDF() { ... }
// 4 razões para mudar!
}

✅ Correto

// Cada classe: uma responsabilidade
public class Pedido { void Calcular() {} }
public class PedidoRepository {
void Salvar(Pedido p) {}
}
public class NotificacaoService {
void EnviarEmail(Pedido p) {}
}

OCP e LSP

🔓 OCP — Open/Closed

Aberto para extensão, fechado para modificação.

// Novo tipo de desconto:
// não altere a classe existente,
// implemente a interface
public interface IDesconto {
decimal Calcular(decimal preco);
}
public class DescontoBlackFriday : IDesconto {
public decimal Calcular(decimal p) => p * 0.5m;
}

🐦 LSP — Liskov Substitution

Subclasses devem ser substituíveis pelas classes base sem alterar comportamento.

// Violação LSP:
class Quadrado : Retangulo {
// Quadrado não pode ter largura != altura
// Substituir Retangulo por Quadrado quebra o código!
}

// Prefira composição ou interface separada

ISP e DIP

🔌 ISP — Interface Segregation

Clientes não devem ser forçados a depender de métodos que não usam.

// Interfaces coesas
public interface ILeitor { string Ler(); }
public interface IEscritor { void Escrever(string s); }
// Não uma interface gorda
// interface IArquivo { Ler(); Escrever(); Deletar(); }

💉 DIP — Dependency Inversion

Módulos de alto nível não devem depender de baixo nível. Ambos dependem de abstrações.

// Alto nível depende de abstração
public class PedidoService {
private readonly IEmailService _email;
public PedidoService(IEmailService email) {
_email = email; // injetado
}
}

SOLID: Resumo Prático

📊 Os 5 Princípios em Uma Frase

  • S — SRP: Uma classe, uma razão para mudar.
  • O — OCP: Estenda sem modificar código existente.
  • L — LSP: Subclasses substituem a base sem surpresas.
  • I — ISP: Interfaces pequenas e coesas.
  • D — DIP: Dependa de abstrações, não implementações.

🎯 Resultado

Código mais testável, flexível, com menor acoplamento e maior coesão. Cada mudança tem impacto previsível e localizado.
