Como o SQL funciona?

🧪 Parser e Planner

  • O banco recebe a query como texto
  • Parser verifica sintaxe e semântica
  • Planner/Optimizer escolhe o melhor plano de execução
  • Executor percorre os dados e retorna resultado

🧭 Categorias SQL

  • DDL: CREATE, ALTER, DROP
  • DML: SELECT, INSERT, UPDATE, DELETE
  • DCL: GRANT, REVOKE
  • TCL: BEGIN, COMMIT, ROLLBACK

DDL e DML na Prática

🏗️ DDL — Estrutura

CREATE TABLE pedidos (
id SERIAL PRIMARY KEY,
cliente_id INT NOT NULL,
total DECIMAL(10,2),
criado_em TIMESTAMP DEFAULT NOW()
);

📝 DML — Dados

-- Inserir
INSERT INTO pedidos (cliente_id, total)
VALUES (1, 150.00);

-- Consulta com JOIN
SELECT p.id, c.nome, p.total
FROM pedidos p
JOIN clientes c ON c.id = p.cliente_id;

Performance: Índices e Planos

⚡ Índices

  • Estrutura B-Tree: busca em O(log n)
  • Colunas em WHERE, JOIN e ORDER BY
  • Índices compostos para queries combinadas
CREATE INDEX idx_pedidos_cliente ON pedidos(cliente_id);

🔍 Execution Plan

  • Seq Scan: leitura completa da tabela
  • Index Scan: usa índice para filtrar
  • Hash Join vs Nested Loop
EXPLAIN ANALYZE SELECT * FROM pedidos WHERE cliente_id = 1;

Transações ACID com TCL

O que garante ACID?

Atomicidade: tudo ou nada | Consistência: estado válido sempre | Isolamento: transações não interferem | Durabilidade: persistência garantida

BEGIN;
UPDATE contas SET saldo = saldo - 100 WHERE id = 1;
UPDATE contas SET saldo = saldo + 100 WHERE id = 2;
-- Se algo falhar:
ROLLBACK;
-- Se tudo ok:
COMMIT;

Perigo: SQL Injection

☠️ Ataque Clássico

O atacante insere SQL malicioso no input do usuário para alterar a query original.

// Input malicioso: ' OR '1'='1
string query = "SELECT * FROM users WHERE nome = '" + input + "'";
// Resultado: SELECT * FROM users WHERE nome = '' OR '1'='1'
// Retorna TODOS os usuários!

🛡️ Solução: Parameterized Queries com Dapper

var user = conn.QueryFirstOrDefault<User>(
"SELECT * FROM users WHERE nome = @Nome",
new { Nome = input } // Parâmetro seguro
);

Dapper: CRUD em .NET

🔌 O que é Dapper?

Micro-ORM para .NET que estende IDbConnection. Mapeia resultados SQL diretamente para objetos C# com alta performance.

🛠️ Operações Básicas

// Query
var pedidos = conn.Query<Pedido>(
"SELECT * FROM pedidos WHERE total > @Min",
new { Min = 100 });

// Execute
conn.Execute(
"INSERT INTO pedidos (total) VALUES (@Total)",
new { Total = 250.00 });
