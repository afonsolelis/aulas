O que é Domínio?

📖 Definição

Domínio é o problema de negócio que o software precisa compreender e operar. É a área de conhecimento especializado que dá sentido ao sistema.

🏥 Saúde

Consultas, diagnósticos, prontuários, prescrições, planos de saúde.

🛒 E-commerce

Catálogo, pedidos, pagamentos, estoque, entrega, devoluções.

🏦 Financeiro

Contas, transferências, investimentos, compliance, auditoria.

Linguagem Ubíqua

"Cliente" significa o mesmo para todos?
No setor de vendas: quem compra. No suporte: quem abre chamado. Na fatura: quem paga. Se a linguagem muda, o modelo precisa mudar.

✅ Com Linguagem Ubíqua

  • Time e negócio usam os mesmos termos
  • Código reflete o vocabulário do domínio
  • Reuniões de requisitos mais produtivas
  • Bugs de nomenclatura eliminados

❌ Sem Linguagem Ubíqua

  • Mesmo objeto, nomes diferentes em camadas
  • Tradução constante gera erros silenciosos
  • Código desconectado do negócio
  • Onboarding difícil

Bounded Context: Fronteiras Claras

🗺️ Por que um modelo único falha?

Sistemas grandes possuem diferentes times, terminologias e regras. Um modelo global único gera conflito de significado e acoplamento desnecessário.

Contexto de Vendas

Pedido tem itens, preços, desconto, status de pagamento. Entidade rica com regras de negócio de vendas.

Contexto de Logística

Pedido tem endereço de entrega, peso, rastreio. Sem preço — outro significado, outro modelo.

Agregados e Raízes em .NET

🧩 O que é um Agregado?

  • Cluster de entidades com invariantes compartilhadas
  • Raiz de Agregado: ponto de entrada único
  • Referências externas somente pela raiz
  • Transação por agregado

💻 Raiz de Agregado

public class Pedido { // Raiz
private readonly List<ItemPedido> _itens = new();
public IReadOnlyCollection<ItemPedido> Itens
=> _itens.AsReadOnly();

public void AdicionarItem(Produto produto, int qtd) {
if (qtd <= 0) throw new InvalidOperationException();
_itens.Add(new ItemPedido(produto, qtd));
}
}
