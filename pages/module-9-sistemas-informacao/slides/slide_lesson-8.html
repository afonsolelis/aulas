O que é CI (Continuous Integration)

🔁 Princípios do CI

  • Integrar código frequentemente (várias vezes ao dia)
  • Build automático a cada push
  • Testes automatizados no pipeline
  • Falha rápida: detectar problemas cedo

📄 YAML GitLab CI

stages: [build, test, deploy]

build:
stage: build
script:
- dotnet restore
- dotnet build --no-restore

test:
stage: test
script:
- dotnet test --no-build

Engenharia do Caos

💡 Definição

Disciplina de experimentar um sistema em produção para descobrir fraquezas antes que causem interrupções reais. Popularizado pela Netflix com o Chaos Monkey.

1. Hipótese

Definir o comportamento esperado em condições de falha.

2. Experimento

Injetar falha: latência, erro de rede, queda de serviço.

3. Observar

Medir métricas, comparar com baseline e corrigir.

Padrões de Resiliência com Polly

🔌 Circuit Breaker

Interrompe chamadas a um serviço com falhas repetidas para evitar cascata de erros.

  • Closed: funcionando normalmente
  • Open: bloqueando chamadas
  • Half-Open: testando recuperação

💻 Polly em .NET

services.AddHttpClient<IPagamentoService>()
.AddPolicyHandler(HttpPolicyExtensions
.HandleTransientHttpError()
.WaitAndRetryAsync(3,
retry => TimeSpan.FromSeconds(Math.Pow(2, retry)))
)
.AddPolicyHandler(HttpPolicyExtensions
.HandleTransientHttpError()
.CircuitBreakerAsync(5, TimeSpan.FromSeconds(30)));

SLO e Observabilidade

📏 SLI / SLO / SLA

  • SLI: Indicador — latência p99, taxa de erros
  • SLO: Objetivo — p99 < 300ms, 99.9% disponível
  • SLA: Acordo contratual com penalidades
