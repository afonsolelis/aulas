Autenticação x Autorização

🪪 Autenticação — Quem é você?

  • Verificação de identidade
  • Credenciais: senha, biometria, token
  • Resultado: identidade confirmada
  • Ex: Login com email + senha

🔓 Autorização — O que você pode fazer?

  • Controle de acesso a recursos
  • Roles e Claims: Admin, Editor, Viewer
  • Resultado: permissão concedida ou negada
  • Ex: Apenas Admin pode deletar usuários
Confundir esses conceitos gera bugs de negócio e falhas graves de acesso. Um usuário autenticado não é necessariamente autorizado a todos os recursos.

JWT — JSON Web Token

🧱 Estrutura do JWT

  • Header: algoritmo de assinatura (HS256)
  • Payload: claims (sub, role, exp)
  • Signature: garante integridade

Formato: header.payload.signature

💻 JWT em .NET

// Program.cs
builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
.AddJwtBearer(o => {
o.TokenValidationParameters = new() {
ValidateIssuer = true,
ValidIssuer = "minha-api",
ValidateLifetime = true,
IssuerSigningKey = new SymmetricSecurityKey(
Encoding.UTF8.GetBytes(secret))
};
});

Por que MFA é Essencial

📊 Contexto

81% das violações de dados envolvem credenciais comprometidas. O elo fraco é humano, não criptográfico.

Algo que você sabe

Senha, PIN, resposta secreta

Algo que você tem

Token TOTP (Authenticator), SMS, chave física

Algo que você é

Biometria: digital, facial, voz

Decisão Arquitetural: Provedor de Identidade

🔑 Keycloak

  • Open source, self-hosted
  • Controle total dos dados
  • Custo operacional maior
  • LDAP/AD integration nativa

☁️ Auth0

  • SaaS gerenciado
  • Setup rápido
  • Custo por MAU
  • Social login fácil

🏗️ Implementação Própria

  • Máximo controle
  • Alto risco de falha
  • Não recomendado em geral
  • Somente casos específicos
