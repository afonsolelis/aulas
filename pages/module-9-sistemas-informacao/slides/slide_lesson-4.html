Pilar 1: Encapsulamento Real

🛡️ O que é Encapsulamento?

  • Ocultar o estado interno do objeto
  • Expor apenas o que é necessário via interface
  • Proteger invariantes de negócio
  • Campos private, métodos public

👾 Exemplo: Tamagotchi

public class Tamagotchi {
private int _energia = 100;
public void Alimentar() {
if (_energia < 100) _energia += 20;
}
public int Energia => _energia;
}

Herança vs Composição

🏗️ Herança — "É um"

  • Subclasse herda comportamento da base
  • Risco: acoplamento rígido
  • Bom para: variações naturais de tipo
class Animal { void Respirar() {} }
class Cachorro : Animal { void Latir() {} }

🧩 Composição — "Tem um"

  • Delega comportamento a objetos internos
  • Maior flexibilidade e testabilidade
  • Preferir composição sobre herança (GoF)
class Pedido {
private readonly ICalculaFrete _frete;
public Pedido(ICalculaFrete frete) {
_frete = frete;
}
}

Princípios SOLID (Intro)

S — Single Responsibility

Uma classe deve ter apenas um motivo para mudar. Separe responsabilidades: persistência, lógica de negócio, apresentação.

O — Open/Closed

Aberto para extensão, fechado para modificação. Use interfaces e polimorfismo em vez de alterar código existente.

Por que SOLID importa?
  • Código mais fácil de testar
  • Mudanças com menos risco
  • Onboarding de novos devs mais rápido
  • Menos bugs em produção

Ciclo Red-Green-Refactor

🔴 Red — Teste Falha

Escreva um teste que define o comportamento desejado. Ele deve falhar inicialmente.

🟢 Green — Faça Passar

Escreva o mínimo de código para o teste passar. Sem preocupação com elegância ainda.

🔵 Refactor — Limpe

Melhore o código mantendo os testes verdes. Elimine duplicações e melhore legibilidade.

Benefício: TDD força design testável e documenta comportamento esperado com código executável.

Exemplo: Calculadora de Frete

// STEP 1 — Red: teste falha
[Fact] public void FreteBasico_DeveSerCalculado() {
var calc = new CalculadoraFrete();
Assert.Equal(15.0, calc.Calcular(peso: 1.0, distancia: 100));
}

// STEP 2 — Green: código mínimo
public class CalculadoraFrete {
public double Calcular(double peso, double distancia)
=> peso * 0.1 * distancia / 100 + 5.0;
}

// STEP 3 — Refactor: extrair constantes, renomear
