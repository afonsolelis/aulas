O que é Refatoração?

📖 Definição (Martin Fowler)

"Refatoração é o processo de modificar um sistema de software de modo que o comportamento externo do código não mude, mas a estrutura interna seja melhorada."

✅ Quando refatorar?

  • Antes de adicionar nova funcionalidade
  • Ao corrigir um bug
  • Durante code review
  • Após testes verdes (TDD)

❌ Quando NÃO refatorar?

  • Sem cobertura de testes
  • Em deadline crítico de entrega
  • Código que será descartado
  • Sem entender o comportamento atual

Code Smells: Sinais de Alerta

📏 Método Longo

Métodos com mais de 20 linhas geralmente têm mais de uma responsabilidade. Extraia submétodos.

🔀 Código Duplicado

Mesma lógica em múltiplos lugares. Uma mudança obriga múltiplas alterações. Extract Method ou superclasse.

🏗️ Classe Deus

Uma classe que faz tudo. Alta coesão e baixo acoplamento são o objetivo. Divida com SRP.

Recurso: Refactoring.Guru — o melhor catálogo visual de code smells e técnicas de refatoração.

Técnicas de Refatoração

🔧 Extract Method

// Antes
void ProcessarPedido() {
// 40 linhas de código
// calcular total + validar + salvar
}

// Depois
void ProcessarPedido() {
decimal total = CalcularTotal();
Validar(total);
Salvar(total);
}

🎭 Replace Conditional with Polymorphism

// Antes (if/switch por tipo)
decimal CalcFrete(string tipo) {
if (tipo == "expresso") return 50;
if (tipo == "normal") return 20;
}

// Depois (polimorfismo)
interface IFrete { decimal Calcular(); }
class FreteExpresso : IFrete {
public decimal Calcular() => 50;
}
