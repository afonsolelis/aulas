Objetivo da Aula

Preparar ambiente de desenvolvimento, criar o primeiro projeto Blazor e iniciar discovery técnico de dados em arquivos e banco.

1) Checklist de Setup

  • Instalar .NET SDK (não só runtime).
  • Confirmar versão com dotnet --version.
  • Instalar VS Code/Visual Studio + extensões C#.
  • Instalar Docker para ambiente de serviços auxiliares.

2) SDK vs Runtime

  • Runtime: executa aplicações já compiladas.
  • SDK: inclui runtime + ferramentas para criar/compilar/testar.
dotnet --info

dotnet new blazor -n Projeto9Dashboard
cd Projeto9Dashboard
dotnet run

3) Blazor WebApp e SSR

O modelo padrão da aula usa renderização no servidor (SSR), com foco em produtividade e integração com stack .NET.

  • UI em componentes reutilizáveis.
  • Backend e frontend em C#.
  • Boa base para dashboards corporativos.

4) Anatomia do Projeto

  • Program.cs: bootstrap da aplicação.
  • Components/Pages: telas e roteamento.
  • wwwroot: assets estáticos.
  • appsettings*.json: configurações por ambiente.

5) Discovery de Dados (Arquivos e DB)

  • Entender estrutura dos dados antes de modelar telas e domínio.
  • Identificar chaves, cardinalidades e lacunas de qualidade.
  • Mapear campos críticos para indicadores do dashboard.
-- exemplo de exploração inicial
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'grupoX';

6) Entregável da Aula 1

  • Ambiente funcional com projeto Blazor rodando localmente.
  • Checklist pós-instalação validado em grupo.
  • Primeiro inventário de dados (fontes, tabelas/campos e hipóteses).
