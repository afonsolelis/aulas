Objetivo da Aula

Consolidar fundamentos de orientação a objetos aplicados em sistemas de informação: modelagem com classes, especialização por herança, contratos por interfaces e organização arquitetural via MVC.

1) Classes e Instâncias

  • Classe: molde com estado e comportamento.
  • Instância: objeto concreto em memória.
  • Separar responsabilidades para evitar classes monolíticas.
class Cliente {
  public string Nome { get; set; }
  public void AtualizarCadastro() { /* ... */ }
}

var cliente = new Cliente();

2) Herança e Polimorfismo

  • Herança para compartilhamento de estrutura/comportamento comum.
  • Polimorfismo para tratar tipos especializados via abstração comum.
  • Usar herança com critério para evitar acoplamento excessivo.
abstract class DocumentoFiscal {
  public string Numero { get; set; }
  public abstract void Emitir();
}

class NotaFiscal : DocumentoFiscal {
  public override void Emitir() { /* ... */ }
}

3) Interfaces e Classes Abstratas

  • Interface: contrato sem implementação obrigatória.
  • Abstrata: base comum com implementação parcial.
  • Combinação das duas melhora extensibilidade e testabilidade.
public interface IRepositorio {
  T ObterPorId(int id);
  void Salvar(T entidade);
}

4) UML de Classes e Relações

  • Relações de associação, agregação e composição para explicitar dependências.
  • Hierarquias de herança para mapear especializações do domínio.
  • UML como linguagem de alinhamento entre negócio e engenharia.

5) MVC em C#

  • Model: regras e dados do domínio.
  • View: interface e apresentação.
  • Controller: orquestra fluxo de requisição/resposta.

No contexto corporativo, MVC organiza responsabilidades e facilita manutenção evolutiva.

6) Arquitetura empresarial com classes

  • Camadas de domínio, aplicação e infraestrutura.
  • Uso de DTOs, serviços e repositórios para desacoplamento.
  • Modelos consistentes para ERP e sistemas de informação.
abstract class EntidadeBase {
  public int Id { get; set; }
  public DateTime DataCriacao { get; set; }
}

class Produto : EntidadeBase {
  public string Nome { get; set; }
  public decimal Preco { get; set; }
}

7) Boas práticas (SOLID + design)

  • SRP: classes com responsabilidade única.
  • OCP: extensão sem alteração do núcleo.
  • LSP/ISP/DIP para preservar consistência e baixo acoplamento.
  • Nomes claros, métodos pequenos e interfaces relevantes.

8) Entregável da Aula 6

  • Modelo de classes revisado com relações UML coerentes.
  • Estrutura MVC mínima validada no projeto.
  • Refino de design orientado a princípios SOLID.
