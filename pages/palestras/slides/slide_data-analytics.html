O que é Ciência de Dados?
Parte 1 — Fundamentos

Definição

Ciência de Dados é um campo interdisciplinar que usa métodos científicos, algoritmos e sistemas para extrair conhecimento e insights a partir de dados estruturados e não estruturados.

Matemática e Estatística

Probabilidade, inferência, regressão, otimização. A base formal para modelar problemas.

Computação

Programação, bancos de dados, engenharia de software. A infraestrutura para processar dados em escala.

Conhecimento de Domínio

Entender o negócio, formular as perguntas certas e traduzir resultados em ação.

Diagrama de Venn de Drew Conway: Data Science vive na interseção dessas três áreas. Sem domínio, você tem modelos sem contexto. Sem computação, você tem teoria sem execução. Sem estatística, você tem código sem rigor.

O que é Análise de Dados?
Parte 1 — Fundamentos

Definição

Análise de Dados é o processo de inspecionar, limpar, transformar e modelar dados com o objetivo de descobrir informações úteis, apoiar conclusões e suportar a tomada de decisão.

Foco principal

  • Responder perguntas de negócio com dados existentes
  • Criar relatórios, dashboards e KPIs
  • Identificar tendências e padrões
  • Comunicar insights de forma clara

Ferramentas comuns

  • SQL — consultar bancos de dados
  • Excel / Google Sheets — análises rápidas
  • Power BI / Tableau / Metabase — visualização
  • Python / R — análises programáticas

Ponto-chave: O analista de dados não precisa criar modelos de Machine Learning. Seu valor está em transformar dados brutos em narrativas que direcionam decisões.

Não é hierarquia: Análise de dados não é "menos" que ciência de dados. São disciplinas complementares. Muitas empresas precisam mais de bons analistas do que de cientistas de dados.

O Ecossistema de Dados nas Empresas
Parte 1 — Fundamentos

De onde vêm os dados?

Toda empresa gera dados continuamente — transações, logs, interações de usuário, sensores, redes sociais. O desafio é transformar esse volume bruto em decisão.

Dados Estruturados

  • Tabelas em bancos relacionais (SQL)
  • Planilhas e CSVs
  • Campos bem definidos (nome, data, valor)
  • Fáceis de consultar e agregar

Dados Não Estruturados

  • Textos livres (e-mails, reviews)
  • Imagens e vídeos
  • Logs de servidor e IoT
  • Requerem processamento especializado

Volume de dados no mundo

Em 2025, estima-se que o mundo gere 181 zettabytes de dados. Aproximadamente 80-90% são não estruturados. Empresas que sabem usar seus dados têm vantagem competitiva mensurável.

Em empresas menores, uma pessoa frequentemente acumula 2-3 desses papéis. O importante é entender onde cada competência entra.

Os 4 Tipos de Analytics
Parte 2 — Maturidade Analítica

A maturidade analítica de uma organização pode ser medida pelo tipo de pergunta que ela consegue responder com dados. Cada nível agrega mais valor — e mais complexidade.

Descritiva

"O que aconteceu?"

Relatórios, dashboards, KPIs históricos.

Diagnóstica

"Por que aconteceu?"

Drill-down, correlações, análise de causa raiz.

Preditiva

"O que vai acontecer?"

Modelos estatísticos, ML, séries temporais.

Prescritiva

"O que fazer?"

Otimização, simulação, recomendação automática.

Escada de valor: Descritiva → Diagnóstica → Preditiva → Prescritiva. Cada degrau aumenta o valor gerado, mas também a complexidade técnica e a exigência de dados de qualidade.

Analytics Descritiva
Parte 2 — "O que aconteceu?"

Objetivo

Resumir dados históricos para entender o estado atual e passado do negócio. É o tipo mais comum e a base de toda análise.

Técnicas

  • Agregações: soma, média, contagem, mediana
  • Segmentação por período, região, produto
  • Tabelas dinâmicas e pivots
  • KPIs e métricas de acompanhamento

Exemplos práticos

NegócioPergunta descritiva
VarejoQual foi o faturamento do mês?
SegurosQuantos sinistros foram abertos no trimestre?
SaaSQual é a taxa de churn mensal?
RHQual o turnover por departamento?

SQL típico

SELECT regiao, COUNT(*) as total_sinistros, AVG(valor) as ticket_medio FROM sinistros WHERE ano = 2025 GROUP BY regiao ORDER BY total_sinistros DESC;

Analytics Diagnóstica
Parte 2 — "Por que aconteceu?"

Objetivo

Investigar as causas por trás dos padrões observados na análise descritiva. Vai além do "o quê" para o "por quê".

Técnicas

  • Drill-down: detalhar métricas por dimensões
  • Análise de correlação entre variáveis
  • Comparação de coortes (A vs B)
  • Análise de causa raiz (5 Whys, Ishikawa)

Exemplo aplicado

Situação: O churn aumentou 15% no último trimestre.

Drill-down:

  1. Segmentar por plano → churn concentrado no plano básico
  2. Segmentar por região → Sul e Sudeste mais afetados
  3. Cruzar com NPS → clientes com NPS < 6 cancelaram 3x mais
  4. Cruzar com suporte → 70% dos cancelamentos abriram ticket sem resolução nos últimos 30 dias

Diagnóstico: Falha no atendimento do plano básico nas regiões Sul/Sudeste.

Analytics Preditiva
Parte 2 — "O que vai acontecer?"

Objetivo

Usar dados históricos para construir modelos que estimem eventos futuros. É aqui que Machine Learning entra com força.

Técnicas

  • Regressão linear e logística
  • Árvores de decisão e Random Forest
  • Séries temporais (ARIMA, Prophet)
  • Redes neurais para padrões complexos

Exemplos por setor

SetorPrevisão
SegurosProbabilidade de sinistro por perfil
BancoScore de crédito e inadimplência
E-commerceDemanda de produtos por SKU
SaúdeRisco de readmissão hospitalar
MarketingPropensão de compra por lead

Importante: Modelos preditivos não são bolas de cristal. Eles estimam probabilidades com base em padrões do passado. Se o passado não representa o futuro (ex.: pandemia), o modelo falha.

Analytics Prescritiva
Parte 2 — "O que fazer?"

Objetivo

Não apenas prever, mas recomendar a melhor ação. Combina modelos preditivos com otimização e regras de negócio.

Técnicas

  • Otimização matemática (programação linear)
  • Simulação Monte Carlo
  • Sistemas de recomendação
  • Reinforcement Learning

Exemplos

  • Logística: Qual rota minimiza custo e tempo?
  • Precificação: Qual preço maximiza receita sem perder clientes?
  • Seguros: Qual prêmio oferecer para equilibrar risco e competitividade?
  • Netflix: Qual filme recomendar para maximizar engajamento?

Diferença-chave

Preditiva diz "há 70% de chance de churn". Prescritiva diz "ofereça 20% de desconto no plano anual — isso reduz a chance de churn para 25%".

O Pipeline de Dados
Parte 3 — Do dado bruto ao insight

Todo projeto de dados segue um fluxo. Pular etapas é a principal causa de análises erradas e modelos ruins. O famoso "garbage in, garbage out".

1. Coleta

Identificar fontes, extrair dados, definir frequência. APIs, bancos, scraping, formulários.

2. Limpeza

Tratar nulos, duplicatas, outliers, inconsistências. A etapa que consome 60-80% do tempo.

3. Análise Exploratória

Estatísticas descritivas, distribuições, correlações. Entender o dado antes de modelar.

4. Modelagem / Análise

Aplicar técnicas analíticas, criar modelos, testar hipóteses.

5. Visualização

Comunicar resultados com gráficos, dashboards e narrativa de dados.

6. Ação e Monitoramento

Tomar decisão baseada nos insights. Monitorar métricas ao longo do tempo.

LGPD e Dados

Desde 2020, a Lei Geral de Proteção de Dados regula coleta, armazenamento e uso de dados pessoais no Brasil. Todo projeto de dados precisa considerar consentimento, finalidade e minimização.

Análise Exploratória (EDA)
Parte 3 — Pipeline

O que é EDA?

Análise Exploratória de Dados é a etapa em que você conhece seus dados antes de tirar conclusões. Criada por John Tukey nos anos 1970, é fundamental para evitar vieses e suposições erradas.

Perguntas típicas da EDA

  • Quantas linhas e colunas temos?
  • Qual a distribuição de cada variável?
  • Existem valores ausentes? Onde?
  • Quais variáveis estão correlacionadas?
  • Existem outliers relevantes?
  • Os dados confirmam o que esperávamos?

Comandos essenciais (Pandas)

df.shape — dimensões

df.describe() — estatísticas descritivas

df.info() — tipos e nulos

df.value_counts() — frequências

df.corr() — matriz de correlação

df.hist() — histogramas rápidos

Regra de ouro: Nunca modele sem antes fazer EDA. O modelo mais sofisticado do mundo falha se os dados estão sujos ou enviesados.

MedidaO que mostra

Por que isso importa?

Média de salário no Brasil: ~R$ 3.000. Mas a mediana é ~R$ 1.600. A média é puxada para cima por altos salários. Sem entender dispersão, você toma decisões erradas.

Distribuições e Probabilidade
Parte 4 — Estatística

Distribuição Normal (Gaussiana)

A mais importante em estatística. Forma de sino, simétrica em torno da média.

  • 68% dos dados: dentro de 1 desvio padrão
  • 95% dos dados: dentro de 2 desvios padrão
  • 99.7% dos dados: dentro de 3 desvios padrão

Exemplos: alturas, notas de prova, erros de medição.

Outras distribuições importantes

DistribuiçãoUso
BinomialSim/não com probabilidade fixa (clicou ou não)
PoissonContagem de eventos raros (sinistros por mês)
ExponencialTempo entre eventos (tempo até falha)
UniformeTodos os valores igualmente prováveis

Probabilidade no dia a dia

Seguros: A distribuição de Poisson modela quantos sinistros esperar por mês. A exponencial modela o tempo entre sinistros. São a base matemática para calcular prêmios e reservas.

Correlação vs Causalidade
Parte 4 — Estatística

Correlação

Mede a força e direção da relação linear entre duas variáveis. Varia de -1 a +1.

ValorInterpretação
+0.7 a +1.0Correlação positiva forte
+0.3 a +0.7Correlação positiva moderada
-0.3 a +0.3Correlação fraca ou inexistente
-0.7 a -0.3Correlação negativa moderada
-1.0 a -0.7Correlação negativa forte

Causalidade

Causalidade implica que A causa B. Correlação apenas diz que A e B variam juntos.

Correlações espúrias famosas

  • Consumo de sorvete correlaciona com afogamentos (ambos aumentam no verão)
  • Vendas de guarda-chuva correlacionam com acidentes de carro (ambos aumentam com chuva)
  • Filmes de Nicolas Cage correlacionam com afogamentos em piscina

A variável confundidora (confounding variable) é a causa real por trás de ambas.

SQL — A Linguagem dos Dados
Parte 4 — Ferramentas

Por que SQL é essencial?

SQL existe desde 1974 e continua sendo a habilidade #1 exigida em vagas de dados. Todo banco relacional fala SQL. Todo data warehouse fala SQL. Todo profissional de dados precisa de SQL.

Comandos fundamentais

SELECT — o que você quer ver

FROM — de onde vêm os dados

WHERE — filtros

GROUP BY — agrupamentos

HAVING — filtro após agrupamento

ORDER BY — ordenação

JOIN — combinar tabelas

Exemplo real

"Top 5 produtos com mais reclamações nos últimos 90 dias":

SELECT produto, COUNT(*) as total FROM reclamacoes WHERE data >= CURRENT_DATE - INTERVAL '90 days' GROUP BY produto ORDER BY total DESC LIMIT 5;

Dica: Aprenda SQL antes de Python. SQL resolve 70% dos problemas de análise de dados e é muito mais rápido para consultas em bancos de dados do que carregar tudo em memória com Pandas.

Machine Learning — Visão Geral
Parte 4 — Ferramentas

O que é Machine Learning?

Subárea da IA em que algoritmos aprendem padrões a partir de dados sem serem explicitamente programados para cada regra. O modelo melhora com mais dados e feedback.

Supervisionado

Aprende com exemplos rotulados.

  • Classificação: spam ou não spam
  • Regressão: prever preço de imóvel

Não Supervisionado

Encontra padrões sem rótulos.

  • Clustering: segmentar clientes
  • Redução: PCA, t-SNE

Por Reforço

Aprende por tentativa e erro.

  • Jogos: AlphaGo
  • Robótica: navegação

Algoritmos mais usados na prática

Regressão Linear · Regressão Logística · Árvore de Decisão · Random Forest · XGBoost · K-Means · KNN

Cultura Data-Driven
Transformação organizacional

O que significa ser data-driven?

Não é ter um time de dados. É uma cultura onde decisões em todos os níveis são informadas por evidências, não apenas por intuição ou hierarquia.

Pilares

  • Acessibilidade: dados disponíveis para quem precisa
  • Literacia: pessoas sabem interpretar dados
  • Governança: dados confiáveis e consistentes
  • Ação: insights viram decisões reais

Antipadrões comuns

  • "Eu acho que..." sem olhar os dados
  • Dashboard bonito que ninguém consulta
  • Dados em silos, cada área com sua "verdade"
  • Relatório de 50 páginas que ninguém lê
  • Investir em IA sem ter dados limpos

Maturidade analítica não é tecnologia: Empresas com planilhas bem organizadas e cultura de decisão baseada em dados são mais maduras que empresas com Spark cluster e zero governança.
