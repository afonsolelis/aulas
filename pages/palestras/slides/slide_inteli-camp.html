Inteli Camp

Trilha Tech · construindo um agente de IA no Lovable em 5 dias

🎯 O que vamos construir

Em 5 dias, cada grupo sai do zero a um MVP publicado: um agente de IA, feito no Lovable, que resolve o desafio [marca/organização parceira] gerando [tipo de output] para [público-alvo].

  • Metodologia PBL (mão na massa em grupo)
  • Trilhas Tech + Empreendedorismo em times multidisciplinares
  • Rotina ágil: dailies e kanban
  • Entrega parcial todo dia, demo na sexta

🛠️ Stack da trilha Tech

  • Lovable — app + agente + publicação
  • Integração com APIs de IA
  • Engenharia de prompts parametrizável
  • Deploy por link público

Dia 1 — Fundamentos & Primeiro Agente

Do zero a um chat publicado no Lovable

Dia 1 · Tech

📚 Conteúdo

  1. Abertura: o desafio [marca] + metodologia Inteli (PBL)
  2. Rotina ágil: formação de grupos, papéis, dailies e kanban
  3. Lovable: projeto, páginas, componentes e fluxo de conversa
  4. Configurar o agente: persona, escopo, tom, limites e mensagens de fallback
  5. Conectar front ao agente (inputs, histórico, loading/erro)
  6. Publicar: gerar URL e validar acesso externo

🎯 Entregável do dia

Entrega
Chat MVP publicado: UI mínima (campo de texto, botão enviar, lista de mensagens), funcionando fora do ambiente local e acessível por link público.
🟣 Lovable 🤖 Persona do agente 🔗 Link público

Dia 2 — APIs & Regra de Negócio

Conectar o agente a um serviço externo e dar-lhe uma lógica

Dia 2 · Tech

📚 Conteúdo

  1. Conceitos: endpoints, métodos (GET/POST), headers, auth, payload, status codes
  2. Montar requisições e tratar respostas/erros (timeouts, retries, mensagens amigáveis)
  3. Criar uma regra de negócio mensurável (validação, cálculo, classificação, roteamento…)
  4. Testes manuais + checklist (cenário feliz + cenários de falha)

🎯 Entregável do dia

Entrega
API integrada (real ou mock) com 1 regra de negócio comprovadamente funcionando, documentada com exemplos de entrada/saída.
🔌 API 🧮 Regra de negócio ✅ Checklist de testes

Dia 3 — Prompts Parametrizáveis

Tornar o agente flexível, reutilizável e controlável

Dia 3 · Tech

📚 Conteúdo

  1. Estrutura do prompt: instruções fixas + variáveis + contexto
  2. Parametrização: objetivo, público, tom, formato de saída, idioma, restrições
  3. UI: formulário para o usuário preencher parâmetros (validação + exemplos)
  4. Biblioteca de templates de prompt e seleção
  5. Boas práticas: evitar prompt injection, limites do usuário, padrões de resposta
  6. Início do protótipo funcional (traduzir a UX de papel → Lovable)

🎯 Entregável do dia

Entrega
Agente flexível e reutilizável: prompts dinâmicos via parâmetros e/ou um espaço onde o usuário configura o próprio prompt conforme a necessidade.
✍️ Prompt parametrizado 🧱 Templates 🛡️ Anti-injection

Dia 4 — Output End-to-End

Do pedido do usuário ao [output] entregue na tela

Dia 4 · Tech

📚 Conteúdo

  1. Definir o caso de uso do output ([output] — ex.: imagens, textos, recomendações)
  2. Estrutura do prompt para o output (estilo, formato, restrições)
  3. Pipeline: pedido → processamento → retorno → renderização no front
  4. Controles do usuário (opções, variações, histórico)
  5. Estados (gerando… / pronto / falhou) + tratamento de erro e latência

🎯 Entregável do dia

Entrega
MVP publicável end-to-end: o usuário descreve o que quer e recebe [output], com UI que exibe resultados, histórico simples e opção de refazer.
🔄 Pipeline completo 🎛️ Controles do usuário ⏳ Estados & erros

Dia 5 — Deploy & Demo

Fechar o produto e apresentar ao vivo

Dia 5 · Tech

📚 Conteúdo

  1. Deploy e ajustes finais para a demonstração ao vivo
  2. Checklist de robustez: o MVP funciona sem você do lado?
  3. Apoio ao pitch (integração com a trilha de Empreendedorismo)
  4. Rodada de apresentações + feedback orientado

🎯 Entregável do dia

Entrega
Demonstração final do protótipo ao vivo + pitch da solução. Rodadas de ~10 min (7 min de fala + feedback).
🚀 Deploy 🎤 Pitch 🗣️ Feedback
