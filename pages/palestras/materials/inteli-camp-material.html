❄️ Sobre o Inteli Camp — Trilha Tech

O Inteli Camp é o bootcamp de inverno do Inteli (julho de 2026): uma imersão de 5 dias em metodologia PBL, onde grupos multidisciplinares (Tech + Empreendedorismo) resolvem um problema real com IA. Este material cobre a trilha Tech, na qual cada grupo constrói — do zero ao deploy — um agente de IA no Lovable que gera [tipo de output] para resolver o desafio de [marca/organização parceira].

🎯 O desafio (case)

Briefing

  • Parceiro: [marca/organização parceira]
  • Contexto: [contexto do produto ou serviço]
  • Problema: [descreva o problema real que os alunos vão resolver com IA]
  • Output do agente: [tipo de output] para [público-alvo]

🎓 Objetivos de aprendizagem (Tech)

  • Compreender fundamentos de IA generativa/agêntica aplicada a um produto.
  • Explorar APIs de IA e técnicas de integração em aplicações web (Lovable).
  • Desenvolver engenharia de prompts controlada e parametrizável.
  • Criar e publicar um protótipo funcional end-to-end.
  • Trabalhar em equipe multidisciplinar com rotina ágil (dailies, kanban).

Dia 1 Fundamentos & Primeiro Agente

Conteúdo

  1. Abertura: desafio [marca] + metodologia Inteli (PBL).
  2. Rotina ágil: formação de grupos, papéis, dailies e kanban.
  3. Lovable: estrutura do projeto (páginas, componentes, fluxo de conversa).
  4. Configuração do agente: persona, escopo, tom, limites e mensagens de fallback.
  5. Conexão do front-end com o agente (inputs, histórico, estados de carregamento/erro).
  6. Publicação/compartilhamento: geração de URL e validação de acesso externo.
Entregável Chat MVP com UI mínima (campo de texto, botão enviar, lista de mensagens), funcionando fora do ambiente local e acessível por link público.
🟣 Lovable🤖 Persona do agente🔗 Link público

Dia 2 APIs & Regra de Negócio

Conteúdo

  1. Conceitos: endpoints, métodos (GET/POST), headers, auth, payload, status codes.
  2. Montagem de requisições e tratamento de respostas/erros (timeouts, retries simples, mensagens amigáveis).
  3. Criação de uma regra de negócio mensurável (validação, cálculo, elegibilidade, priorização, classificação, roteamento).
  4. Testes manuais e checklist de validação (cenários feliz + cenários de falha).
Entregável Integração com API (real ou mock) rodando, com pelo menos 1 regra de negócio implementada e testada de ponta a ponta, documentada com exemplos de entrada/saída.
🔌 API🧮 Regra de negócio✅ Checklist de testes

Dia 3 Prompts Parametrizáveis

Conteúdo

  1. Estrutura de prompt: instruções fixas + variáveis (placeholders) + contexto.
  2. Parametrização: objetivo, público-alvo, tom, formato de saída, idioma, restrições.
  3. UI/UX: formulário para o usuário preencher parâmetros (com validação e exemplos).
  4. Templates: biblioteca de prompts prontos e mecanismo de seleção.
  5. Boas práticas: evitar prompt injection, limites do que o usuário pode alterar, padrões de resposta.
  6. Início do protótipo funcional (traduzir a UX de papel para a estrutura no Lovable).
Entregável Agente flexível e reutilizável: prompts dinâmicos via parâmetros e/ou um espaço onde o usuário configura o próprio prompt conforme a necessidade.
✍️ Prompt parametrizado🧱 Templates🛡️ Anti-injection

Dia 4 Output End-to-End

Conteúdo

  1. Definição do caso de uso do output ([output] — ex.: imagens, textos, recomendações).
  2. Estrutura do prompt para o output (estilo, composição, formato, restrições).
  3. Pipeline: pedido → processamento → retorno → renderização no front-end.
  4. Controles do usuário (opções, variações, histórico de gerações).
  5. Tratamento de filas/latência e estados (gerando… / pronto / falhou) + mensagens de erro.
Entregável MVP publicável com geração de [output] end-to-end: do pedido do usuário ao retorno visível no front-end, com UI que exibe resultados, histórico simples e opção de refazer.
🔄 Pipeline completo🎛️ Controles do usuário⏳ Estados & erros

Dia 5 Deploy & Demo

Conteúdo

  1. Deploy e ajustes finais para a demonstração ao vivo.
  2. Checklist de robustez: o MVP funciona sem você do lado?
  3. Apoio ao pitch em conjunto com a trilha de Empreendedorismo.
  4. Rodada de apresentações + sessão de feedback orientado.
Entregável Demonstração final do protótipo ao vivo (IA gerando [output]) + pitch. Rodadas de ~10 minutos (7 min de apresentação + feedback).
