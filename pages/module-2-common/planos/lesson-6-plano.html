Resumo

A aula 6 fecha o bloco de back-end com a API completa: endpoints de leitura com paginação e filtros, criação com validação de body, atualização (PUT vs. PATCH) e remoção com verificação de existência. Introduzimos middlewares como interceptors (logger, auth, body parser), validação na borda com express-validator e um error handler global que padroniza respostas de erro.

O fechamento foca em testar a API antes do frontend — usando curl e REST Client — e na ideia de que cada endpoint nasce de um Business Driver documentado.

Objetivos de aprendizagem

  • OA-1Implementar endpoints REST completos (GET, POST, PUT, PATCH, DELETE) com status codes corretos.
  • OA-2Diferenciar PUT (substituição total) e PATCH (modificação parcial), aplicando o método correto a cada cenário.
  • OA-3Escrever middlewares próprios e usar middlewares prontos (express.json, cors, logger).
  • OA-4Validar dados de entrada com express-validator antes de chegar no Controller.
  • OA-5Implementar error handler global que captura erros e devolve resposta padronizada.
  • OA-6Testar todos os endpoints da API com REST Client antes do frontend ser tocado.

Pré-requisitos

  • Aula 5 concluída: estrutura MVC com 1 endpoint GET funcional.
  • Banco com massa de teste suficiente para todos os verbos.
  • Cliente HTTP escolhido pela equipe (REST Client, Postman ou curl).

Estratégias de metodologia ativa

  • Decisão fundamentada — escolha entre PUT e PATCH exige defesa antes da implementação.
  • API antes do frontend — todos os endpoints são exercitados via REST Client antes de qualquer linha de HTML/JS.
  • Replicação imediata — cada bloco do professor é seguido por implementação em pares na entidade real do projeto.
Princípio orientador

Resposta de erro nunca expõe stack trace ao cliente. O errorHandler existe para isso.

Critério mínimo

Aluno que sair às 16h sem API CRUD completa testada via REST Client está em débito técnico para iniciar o frontend.

Conexão com a aula 7

A aula 7 abre o front-end. A API construída hoje será o servidor que o navegador vai consumir.

  • O requests.http documenta o contrato — frontend não pode adivinhar shape de JSON.
  • Endpoints precisam estar estáveis na develop antes da aula 7.
Para o autoestudo da aula 7

Antes das 8h: ler material da aula 7, esboçar o HTML da tela principal do projeto e revisar JavaScript ES6 (arrow functions, destructuring, map/filter).
