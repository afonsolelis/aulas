Resumo

A aula 11 fecha o módulo com o que está abaixo do HTTP: pilha TCP/IP vs. modelo OSI, o handshake de 3 vias, DNS resolvendo nomes para IPs em milissegundos, HTTPS como obrigatório em produção, CDNs e load balancers como mecanismos para escalar e ficar perto do usuário. Em cima disso, exploramos padrões de resiliência: retry com exponential backoff, circuit breaker, timeout e os 3 pilares da observabilidade (logs, métricas, traces).

O fechamento amarra os 11 dias do módulo: a equipe escreve uma análise de resiliência do próprio projeto e participa da retrospectiva final.

Objetivos de aprendizagem

  • OA-1Comparar os modelos OSI e TCP/IP e localizar HTTP, DNS e TLS na pilha.
  • OA-2Explicar o handshake de 3 vias do TCP e o papel do DNS na resolução de nomes.
  • OA-3Justificar HTTPS como obrigatório em produção e identificar o conteúdo de um certificado TLS.
  • OA-4Reconhecer CDN e load balancer como soluções de latência e disponibilidade.
  • OA-5Implementar retry com exponential backoff em chamadas HTTP do projeto.
  • OA-6Configurar logs estruturados como base mínima de observabilidade.

Pré-requisitos

  • Aula 10 concluída: pipeline CI verde e testes E2E + integração no projeto.
  • Cliente curl ou similar disponível para inspecionar headers HTTP.
  • Lista de 2 dependências externas que o projeto usa (ou poderia usar).

Estratégias de metodologia ativa

  • Inspeção de produção real — usar sites públicos como objeto de estudo (TLS, CDN headers).
  • Implementação mínima viável de resiliência — não cobrimos tudo, mas todo aluno sai com retry funcionando.
  • Retrospectiva como parte do plano — última hora dedicada a aprender com o caminho percorrido.
Princípio orientador

Sistema sem retry e sem log estruturado é caixa-preta. Você não vai conseguir explicar o que aconteceu quando der ruim.

Fechamento do módulo

O módulo 2 cobriu o ciclo completo: requisitos → banco → API → frontend → testes → resiliência. A equipe sai com:

  • Repositório com Git Flow truncado e Conventional Commits.
  • Banco modelado, queries CRUD/JOIN documentadas e migrações controladas.
  • API MVC com endpoints REST validados, error handler e requests.http.
  • Frontend responsivo, BEM, com CRUD completo e estados de UI.
  • CI verde e testes E2E + integração + cobertura.
  • Operação básica: retry, log estruturado e documento de resiliência.
Próximos passos

Os módulos seguintes aprofundam autenticação, segurança em produção, persistência avançada e arquitetura distribuída. Mantenha o repositório do módulo 2 acessível como referência viva.
