Resumo

A aula 5 abre o bloco de back-end. O aluno entende o que é o Node.js (JavaScript fora do navegador), por que ele é não-bloqueante (Event Loop), e como o Express transforma 10 linhas de código em um servidor HTTP funcional. Em seguida, introduzimos o padrão MVC e implementamos o caminho completo: routes → controllers → models → banco.

O fechamento conecta o back-end ao RM-ODP (visão Computational) e prepara a tarde, em que cada equipe terá um endpoint GET retornando JSON com dados reais do banco do projeto.

Objetivos de aprendizagem

  • OA-1Explicar o papel do Node.js, do Event Loop e do npm na pilha de back-end.
  • OA-2Subir um servidor Express respondendo em http://localhost:3000.
  • OA-3Estruturar o projeto em pastas seguindo o padrão MVC (routes/, controllers/, models/).
  • OA-4Implementar Model que conversa com PostgreSQL via pg e retorna dados consultados.
  • OA-5Conectar Route → Controller → Model → resposta JSON em um endpoint GET.
  • OA-6Mapear a aula no RM-ODP (Computational viewpoint) e justificar a separação de responsabilidades.

Pré-requisitos

  • Node.js LTS instalado e node --version respondendo no terminal.
  • Banco da equipe operacional com massa de teste e queries da aula 4 prontas.
  • Repositório atualizado com a develop no estado pós-aula 4.
  • Variáveis de conexão ao banco anotadas (não commitar credenciais — usar .env).
Nunca commite

Senhas, tokens e .env nunca entram no repositório. Use .env.example com placeholders.

Estratégias de metodologia ativa

  • Predição antes da execução — alunos preveem a ordem do Event Loop antes de rodar o Node.
  • Live coding com replicação imediata — cada bloco do professor é seguido por replicação em pares com variação.
  • Reaproveitamento contextual — Models nascem das queries do dia anterior, evitando código artificial.
Princípio orientador

Nenhuma camada do MVC pode falar a linguagem da outra: rota não tem SQL, model não tem res.json.

Conexão com a aula 6

A aula 6 amplia para o conjunto completo de verbos REST (GET com filtros, POST, PUT, PATCH, DELETE) e introduz middlewares e validação.

  • O Model criado hoje será reaproveitado e ganhará novos métodos de mutação.
  • A pasta backend/ deve estar mergeada na develop antes da próxima daily.
Para o autoestudo da aula 6

Antes das 8h: ler material da aula 6, listar 5 mutações de dados do projeto e estudar diferença entre PUT e PATCH.
