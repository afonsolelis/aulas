➕ INSERT INTO — Inserindo Dados
Adicionando registros à tabela
postgresql — INSERT INTO usuarios
ORDEM
Ordem das colunas importa
Os nomes das colunas devem corresponder à ordem dos valores em VALUES.
VALUES
VALUES deve bater com a ordem
Cada posição em VALUES corresponde a uma coluna listada no INSERT.
RETURNING
RETURNING id retorna o id gerado
Use RETURNING id para obter o ID do registro recém-inserido.
INSERT INTO ... RETURNING id;
🔍 SELECT — Lendo Dados
Consultando registros com precisão
postgresql — SELECT usuarios
SELECT *
Traz TUDO — mais pesado
SELECT cols
Só o necessário

Prefira SELECT id, nome, email a SELECT * — menos dados trafegam, consulta mais rápida.

WHERE, ORDER BY, LIMIT, OFFSET
Filtrando, ordenando e paginando resultados
postgresql — cláusulas de filtragem
WHERE
Filtra linhas
Suporta: = != > < LIKE BETWEEN IN IS NULL
ORDER BY
Ordena resultados
ASC (padrão, crescente) ou DESC (decrescente).
LIMIT
Limita quantidade
Retorna no máximo N linhas — essencial para listas e paginação.
OFFSET
Pula registros
Com LIMIT implementa paginação: página 2 = LIMIT 10 OFFSET 10.
✏️ UPDATE — Atualizando Dados
⚠️ Sempre use WHERE — ou você atualiza tudo!
postgresql — UPDATE usuarios

✅ Boas práticas

  • Filtrar pelo id quando possível
  • Use RETURNING para verificar o que foi alterado
  • Atualize atualizado_em = NOW() para auditoria

⚠️ Cuidado!

Sem WHERE, você atualiza TODOS os registros da tabela! Confirme a cláusula WHERE antes de executar.

🗑️ DELETE — Removendo Dados
⚠️ Operação irreversível — use com cuidado!
postgresql — DELETE FROM usuarios

✅ Diferença importante

  • DELETE FROM t WHERE ... — remove linhas específicas, com log
  • TRUNCATE TABLE t — limpa TUDO, mais rápido, sem log por linha
  • Foreign keys podem bloquear DELETE se há filhos

⚠️ Atenção máxima!

DELETE sem WHERE remove TUDO! Use TRUNCATE para limpar a tabela. Considere soft delete: coluna ativo = false.

🔒 Transações — ACID
Garantindo integridade em operações complexas
postgresql — BEGIN / COMMIT
⚛️
Atomicidade
tudo ou nada
Ou todas as operações são executadas, ou nenhuma é.
✅
Consistência
dados sempre válidos
O banco sempre vai de um estado válido para outro.
🔒
Isolamento
transações não interferem
Cada transação opera como se fosse a única.
💾
Durabilidade
dados persistem
Após COMMIT, dados sobrevivem a falhas e reinicializações.
⚡ Índices — Performance
Acelerando consultas em tabelas grandes
Sem índice
Seq Scan — 850ms
Com índice
2ms
postgresql — CREATE INDEX

📌 Quando criar índice?

  • Colunas usadas em WHERE frequentemente
  • Colunas de JOIN entre tabelas
  • Colunas de ORDER BY em grandes tabelas

⚠️ Custo vs Benefício

Índices aceleram leituras mas custam em escritas. Não indexe todas as colunas.
