O que é um Banco de Dados?
Por que não basta uma planilha?

📁 O que é?

Coleção organizada de dados persistentes, estruturados de forma que possam ser acessados, gerenciados e atualizados com eficiência.

🤔 Por que precisamos?

  • Múltiplos usuários simultâneos
  • Integridade e consistência dos dados
  • Consultas complexas e rápidas
  • Persistência, backup e recuperação

📱 Exemplos do dia a dia

Instagram (posts/likes), Spotify (playlists), iFood (pedidos), sistemas bancários, cadastros de alunos no Inteli.

🔐 Gerencia acesso

Controla quem pode ler, escrever ou administrar cada objeto. Autenticação e autorização por usuário e role.

✅ Garante integridade

Constraints (PK, FK, NOT NULL, UNIQUE) impedem que dados inválidos entrem no banco.

🔄 Controla transações

ACID: Atomicidade, Consistência, Isolamento, Durabilidade. Ou tudo funciona, ou nada é salvo.

📦 Faz backups

Recuperação em caso de falha com WAL (Write-Ahead Log) e snapshots periódicos.

🟦 Entidade

Substantivo — algo com existência própria no domínio do sistema. Ex.: USUARIO, PEDIDO, PRODUTO

🔷 Atributo

Característica de uma entidade. Ex.: nome, email, preço. O atributo id com PK identifica cada linha unicamente.

🔗 Relacionamento

Verbo entre entidades. "USUARIO faz PEDIDO" · "PEDIDO contém PRODUTO". Implementado com Foreign Keys.

Tipos de Dados SQL
Escolha o tipo certo para cada coluna
INTEGER / SERIAL
Números inteiros
SERIAL é auto-increment — ideal para chaves primárias.
id SERIAL PRIMARY KEY
VARCHAR(n) / TEXT
Textos
VARCHAR limita o tamanho. TEXT não tem limite definido.
nome VARCHAR(100)
DECIMAL(p,s)
Decimais precisos
p = total de dígitos, s = casas decimais. Use para dinheiro.
preco DECIMAL(10,2)
BOOLEAN
Verdadeiro / Falso
Aceita true/false ou 1/0. Útil para flags de estado.
ativo BOOLEAN DEFAULT true
TIMESTAMP / DATE
Data e hora
TIMESTAMP inclui hora. DATE é apenas data. NOW() retorna o instante atual.
criado_em TIMESTAMP
JSON / JSONB
Dados semi-estruturados
JSONB é indexável e mais rápido. Útil para metadados flexíveis.
config JSONB
Constraints — Regras de Integridade
O banco garante que só dados válidos entram
PRIMARY KEY
Identifica unicamente cada linha
NOT NULL
Campo obrigatório
UNIQUE
Valor não pode repetir
FOREIGN KEY
Liga tabelas, garante integridade referencial
DEFAULT
Valor padrão quando não informado

💡 Por que constraints importam?

Elas garantem que a aplicação não precise validar tudo manualmente. O banco rejeita automaticamente dados inválidos, protegendo a integridade mesmo com bugs no código.
