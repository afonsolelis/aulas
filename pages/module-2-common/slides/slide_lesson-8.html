Callbacks → Promises → async/await
A evolução do JavaScript assíncrono ao longo dos anos
📜
Callbacks — 2009
buscar(id, function(user) { salvar(user, function(ok) { notificar(ok, function() { ... }) }) })
Problema: "Callback Hell" — pirâmide da perdição, difícil de manter e debugar.
⬇️ ES2015 — Promises chegam ao JavaScript
🤝
Promises — ES2015 (ES6)
buscar(id).then(user => salvar(user)).then(ok => notificar(ok)).catch(err => tratar(err))
Melhor: encadeamento legível, tratamento de erro centralizado com .catch(). Ainda verboso.
⬇️ ES2017 — async/await torna o código ainda mais limpo
⚡
async/await — ES2017
try { const user = await buscar(id); const ok = await salvar(user); await notificar(ok); } catch(err) { tratar(err); }
Melhor de todos: parece código síncrono, usa try/catch nativo, máxima legibilidade.

Tratamento de Erros com fetch()
fetch() não lança erro em respostas 4xx/5xx — você precisa verificar
fetch-errors.js

⚠️ Armadilha do fetch()

fetch() só lança erro em falha de rede (sem conexão, DNS falhou). Para respostas 4xx e 5xx, você precisa verificar res.ok ou res.status manualmente.

🔍 Tipos de erro

  • TypeError — falha de rede, sem conexão
  • 400 — dados inválidos enviados
  • 401 — não autenticado
  • 403 — sem permissão
  • 404 — recurso não encontrado
  • 500 — erro interno do servidor

Loading States e UX
Gerenciar estado da interface durante requisições assíncronas
loading-state.js

🎯 Os 3 estados

  • carregando — mostra spinner, desabilita botões
  • erro — exibe mensagem amigável, permite retry
  • sucesso — renderiza os dados na tela

💡 Boas práticas de UX

  • Sempre mostrar feedback visual de carregamento
  • Nunca deixar a tela travada sem explicação
  • Mensagens de erro legíveis para o usuário
  • Usar finally para limpar o estado
localStorage e sessionStorage
Persistindo dados no navegador sem banco de dados
storage.js

📊 localStorage vs sessionStorage

  • localStorage — persiste entre sessões e abas, sem expiração
  • sessionStorage — dura só enquanto a aba está aberta
  • Ambos armazenam apenas strings — use JSON.stringify/parse
  • Limite de ~5MB por origem

🔒 Segurança

NUNCA guarde senhas no localStorage. Para tokens JWT, prefira httpOnly cookies em produção. O localStorage é acessível por qualquer JS da página (risco XSS).
