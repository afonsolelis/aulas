🚀 Hoje completamos todos os endpoints CRUD da nossa API

Na Aula 5 vimos a estrutura base: Node.js, Express, Models e Controllers. Hoje implementamos os 4 tipos de endpoint — leitura (GET), criação (POST), atualização (PUT/PATCH) e remoção (DELETE) — além de validação e tratamento de erros.

GET — Endpoint de Leitura
Buscar dados com suporte a paginação e filtros por query string
routes/usuarios.js + controllers/usuario.js
HTTP Response — GET /api/usuarios?pagina=1&limite=2
POST — Endpoint de Criação
Receber dados no body, validar e persistir no banco
controllers/usuario.js — async criar(req, res)

📋 Códigos de status para POST

201 Created para sucesso · 400 Bad Request para dados inválidos · 409 Conflict para conflito (ex: email duplicado)

PUT e PATCH — Atualização
PUT substitui o recurso inteiro — PATCH modifica apenas os campos enviados
PUT /api/usuarios/:id — substituir tudo
PATCH /api/usuarios/:id — atualizar parcialmente

🔑 Diferença fundamental

PUT = substituir o recurso inteiro (campos não enviados ficam nulos) · PATCH = modificar apenas os campos enviados (merge parcial)

DELETE — Remoção
Verificar existência antes de deletar e retornar 204 No Content
controllers/usuario.js — async deletar(req, res)

📌 204 No Content

204 No Content = sucesso sem corpo de resposta. Use quando a ação foi concluída mas não há dados para retornar. O body deve estar vazio — use res.status(204).send().

Error Handling Global
Um middleware especial que captura todos os erros da aplicação
middlewares/errorHandler.js

⚠️ 4 parâmetros obrigatórios: (err, req, res, next)

O Express identifica middlewares de erro pela assinatura de 4 argumentos. Deve ser o último middleware registrado em server.js.
