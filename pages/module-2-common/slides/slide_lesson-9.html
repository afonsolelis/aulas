CSS Variables
Tokens de design reutilizáveis — troque uma linha e mude tudo

🎯 Por que usar variáveis CSS?

  • Defina uma vez, use em qualquer lugar
  • Troque o tema inteiro mudando só o :root
  • Funciona em cascata — filhos herdam do pai
  • Compatível com todos os navegadores modernos

🌗 Tema escuro em 3 linhas

Basta redefinir as variáveis com [data-theme="dark"] e o JavaScript alterna o atributo no document.body.

CSS BEM — Naming Convention
Block · Element · Modifier — CSS previsível e escalável
🧱 Block — entidade autônoma
.card
🔩 Element — parte do block (__)
.card__titulo .card__imagem .card__botao
🎨 Modifier — variação do block/element (--)
.card--destaque .card__botao--primario

✅ Regra de ouro BEM

Nunca aninhe seletores: .card .titulo é frágil. Use .card__titulo — sempre plano, sempre previsível.

⏱️ Debounce

Evita que uma função seja chamada repetidamente em eventos frequentes como keyup, scroll ou resize. Só executa após o usuário parar por X ms.

Exemplo real: busca em tempo real ao digitar — sem debounce dispara 1 request por tecla!

📦 Module Pattern (IIFE)

Encapsula estado e lógica em um objeto com interface pública. Evita poluição do escopo global e organiza o código como um mini-módulo.

UsuarioUI.carregar() — interface limpa, implementação oculta.

  • CSS crítico não inline — atrasa First Paint
  • Fontes externas sem font-display: swap

🔴 Spec do debounce

jest.useFakeTimers();
it('executa fn 1x após delay', () => {
  const m = jest.fn();
  const fn = debounce(m, 300);
  fn(); fn(); fn();
  jest.runAllTimers();
  expect(m).toHaveBeenCalledTimes(1);
});

🟢 Green + Refactor

function debounce(fn, ms) {
  let t;
  return (...a) => {
    clearTimeout(t); t = setTimeout(()=>fn(...a), ms);
  };
} // ✅
