Anatomy of a good prompt

Six parts that make the instruction clear and verifiable

Prompts · Concept

🔧 The 6 parts

  1. Role: who the AI is in this context (e.g., an impartial HR analyst)
  2. Context: evaluation criteria and job profile
  3. Task: what to do — a clear verb, no ambiguity
  4. Output format: how to return it (table, JSON)
  5. Constraints: what it must NOT do — derived from the NFRs
  6. Examples: a sample of what a good answer looks like

📋 Example — spec → prompt

The spec says:
Task: evaluate an interview answer
Criteria: clarity, experience, fit to the profile
Output: score 0–5 per criterion + justification
Constraint: do not consider age, gender or origin
Generated prompt:
You are an impartial HR analyst.
Criteria: [criteria] · Role: [profile].
Evaluate the answer [answer] giving a score 0–5 per criterion, with 1 sentence of justification.
Do not use age, gender or origin. Format: table.

Weak prompt × robust prompt

The same task, two very different results

Worked example

❌ Weak prompt

Evaluate this candidate: [answer]
  • No criteria: the AI decides on its own what matters
  • No format: each answer comes out different
  • No constraints: risk of considering age, gender, origin
  • Result: inconsistent scores subject to bias

✅ Robust prompt

Role + criteria + format (table) + constraints (no protected attributes) + example
  • Always evaluates by the same criteria
  • Standardized output — you can compare candidates
  • Reduces bias and protects the candidate's data
  • Result: predictable, fair and testable

Ask for a structured output

A fixed format turns loose text into data usable in the funnel

Format

🧱 Why structure it

  • Free text is hard to compare across candidates
  • A fixed format (table or JSON) becomes data — it feeds the funnel ranking
  • Always ask for: score per criterion + justification + recommendation
Respond in JSON:
{ "criterion": ..., "score": 0-5,
  "justification": ..., "recommendation": advance|review }

📊 Example output

CriterionScoreJustification
Clarity4Explained with objective examples
Experience3A fit, but little time in the role
Profile5Aligned with the job's competencies
Recommendation
Advance to the next stage — final decision by the recruiter.

Calibrate the AI with examples

Showing examples makes the scoring more consistent

Few-shot

🎯 What calibrating means

  • Without a reference, each evaluation can vary
  • Include in the prompt 1–2 examples of an answer with the "right" score
  • The AI starts to follow the same standard — this is reliability (an NFR)
  • A few well-chosen examples are worth more than many

🧪 Example in the prompt

Example of a 5 score: "answer with concrete data and outcome"
Example of a 2 score: "generic answer, no evidence"

Now evaluate to the same standard: [answer]

The examples set the yardstick. Two similar answers start receiving similar scores.

Responsible AI in recruitment

In an agent that evaluates people, this is not optional

Ethics · NFR

⚠️ Three mandatory safeguards

  • Bias: instruct the AI to ignore age, gender, origin, appearance — evaluate only what relates to the work.
  • Privacy (LGPD): do not expose the candidate's personal data; use only what is necessary for the evaluation.
  • Human decision: the AI recommends, the recruiter decides. Never an automatic rejection.

🔗 Where this fits

  • They are prompt constraints — coming straight from the spec's NFRs (Day 1)
  • They connect to ISO/IEC 25010: security and reliability
  • A good prompt writes this explicitly, does not leave it implicit
Constraints: evaluate only competencies related to the job; do not use sensitive personal data; flag uncertainty instead of making things up.

Iterate with test cases

A prompt is not born ready — it is adjusted with evidence

Iteration

🔁 The cycle

  1. Write v1 with the 6 parts
  2. Test on at least 3 example answers
  3. Compare the output with the spec's acceptance criteria
  4. Locate which part of the prompt caused the deviation
  5. Adjust only that part and test again

🧪 Choose good test cases

  • A strong answer (should score high)
  • A weak answer (should score low)
  • A borderline or ambiguous answer
  • If the AI gets the borderline case wrong, the prompt needs more criteria or an example

Parametrization and common failures

One prompt that serves several job roles — and what usually breaks

Robustness

🎛️ Parametrization

  • Mark what changes with brackets: [role], [criteria], [answer]
  • The same prompt now serves several job roles
  • In Lovable, each parameter becomes a form field
  • Test with different parameters to ensure it stays valid

🔨 Hands-on — write and iterate the prompt

Group sprint · 30-min timer · write, test and adjust

⏱️ Active Learning

🧭 Sprint steps

  1. Write v1 of the prompt with the 6 parts
  2. Ask for structured output (table or JSON)
  3. Test on 3 answers: strong, weak and borderline
  4. Compare with the acceptance criteria and adjust one part at a time

👥 Roles in the group

  • Who writes the prompt (keeps the single version)
  • Who tests with the 3 answers
  • Who records what changed at each iteration

✋ Call the professor when

  • The output does not match the criteria after 2 adjustments
  • The AI shows any sign of bias
  • You do not know which part of the prompt to fix

🏁 Acceptance criteria

Ready when
The 3 answers produce output in the defined format, with coherent scores and no use of protected attributes. If not, the adjustment goes into the prompt — not the criteria.

Checklist — is the prompt robust?

Answer "yes" to everything before considering the prompt ready

Self-assessment
  • Does it have a clear role for the AI?
  • Does it bring the context (criteria and job profile)?
  • Is the task a clear verb, without ambiguity?
  • Does it define the output format (table/JSON)?
  • Does it have constraints for bias and privacy?
  • Was it calibrated with at least one example?
  • Was it tested on 3 answers (strong, weak, borderline)?
  • Did the output stay consistent across the tests?
  • Is it parameterized for other job roles?
  • Does it make clear that the decision is human?
