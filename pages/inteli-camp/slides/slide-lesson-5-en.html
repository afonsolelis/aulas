Pitch script

Four screens — copy this structure and fill it with the group's case

Pitch · Structure

1️⃣ Problem

  • Which partner pain does the solution solve?
  • Who suffers from this problem today?
  • One sentence, no technical jargon

2️⃣ Live demo

  • Show the solution working with real data
  • Input → output, from start to finish
  • Have a plan B (video/screenshots) in case the connection becomes unstable

3️⃣ How the prompt works

  • Explain the logic behind its construction
  • Which parameters control the behavior
  • How the spec's constraints were ensured

4️⃣ Result vs. criteria

  • Revisit the spec's acceptance criteria and show that the output meets them
  • Mention how the group mitigated bias in screening and complied with LGPD
  • Close with the value delivered to the partner

Model script — Azul Linhas Aéreas case

Filled-in example — a reference to adapt to the group's work

Pitch · Example

1️⃣ Problem

  • The recruiters and Azul HR team responsible for candidate selection face slow, manual screening of interviews.
  • High volume of candidates per role and risk of inconsistent evaluations across recruiters.

2️⃣ Live demo

  • The agent takes the interview data and returns the classification and summary of candidates across the stages of the selection funnel.
  • From input (interview) to output (selection funnel), from start to finish.

3️⃣ How the prompt works

  • Evaluation criteria parameterized from the role profile.
  • Structured output and constraints against the use of sensitive candidate data.

4️⃣ Result vs. criteria

  • Selection funnel filled in, meeting the spec's acceptance criteria.
  • Bias in screening mitigated and interview data handled in compliance with LGPD.

What the panel evaluates

Five criteria — prepare an answer for each one

Panel · Criteria
1
Understanding of the problem — does the group understand what the partner needs? Can they explain the problem in one sentence without technical jargon?
2
Functional solution — does the live demo work? Does the generated output make sense for the partner's context? Does it meet the spec's acceptance criteria?
3
Prompt quality — can the group explain why the prompt was built that way? Do the constraints and parameters make sense with the spec?
4
Clarity of the presentation — is the narrative coherent from problem to solution? Does each member know how to talk about what they built?
5
Responsible AI — can the group explain how they mitigated bias in candidate screening and how the solution complies with LGPD when handling interview data?
Rehearsal tip: run the demo with the partner's real data, not generic examples. The panel identifies this difference.
