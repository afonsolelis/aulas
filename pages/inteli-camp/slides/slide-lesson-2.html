Anatomia de um bom prompt

Seis partes que tornam a instrução clara e verificável

Prompts · Conceito

🔧 As 6 partes

  1. Papel: quem a IA é neste contexto (ex.: analista de RH imparcial)
  2. Contexto: critérios de avaliação e perfil da vaga
  3. Tarefa: o que fazer — verbo claro, sem ambiguidade
  4. Formato de saída: como devolver (tabela, JSON)
  5. Restrições: o que NÃO pode — vêm dos RNF
  6. Exemplos: uma amostra do que é uma boa resposta

📋 Exemplo — spec → prompt

A spec diz:
Tarefa: avaliar a resposta de uma entrevista
Critérios: clareza, experiência, aderência ao perfil
Saída: nota 0–5 por critério + justificativa
Restrição: não considerar idade, gênero ou origem
Prompt gerado:
Você é um analista de RH imparcial.
Critérios: [critérios] · Vaga: [perfil].
Avalie a resposta [resposta] dando nota 0–5 por critério, com 1 frase de justificativa.
Não use idade, gênero ou origem. Formato: tabela.

Prompt fraco × prompt robusto

A mesma tarefa, dois resultados muito diferentes

Exemplo trabalhado

❌ Prompt fraco

Avalie esse candidato: [resposta]
  • Sem critérios: a IA decide sozinha o que importa
  • Sem formato: cada resposta sai diferente
  • Sem restrições: risco de considerar idade, gênero, origem
  • Resultado: notas inconsistentes e sujeitas a viés

✅ Prompt robusto

Papel + critérios + formato (tabela) + restrições (sem atributos protegidos) + exemplo
  • Avalia sempre pelos mesmos critérios
  • Saída padronizada — dá para comparar candidatos
  • Reduz viés e protege dados do candidato
  • Resultado: previsível, justo e testável

Peça uma saída estruturada

Um formato fixo transforma texto solto em dado utilizável no funil

Formato

🧱 Por que estruturar

  • Texto livre é difícil de comparar entre candidatos
  • Um formato fixo (tabela ou JSON) vira dado — alimenta o ranking do funil
  • Peça sempre: nota por critério + justificativa + recomendação
Responda em JSON:
{ "criterio": ..., "nota": 0-5,
  "justificativa": ..., "recomendacao": avançar|rever }

📊 Exemplo de saída

CritérioNotaJustificativa
Clareza4Explicou com exemplos objetivos
Experiência3Aderente, mas pouco tempo na função
Perfil5Alinhado às competências da vaga
Recomendação
Avançar para a próxima etapa — decisão final do recrutador.

Calibre a IA com exemplos

Mostrar exemplos deixa a pontuação mais consistente

Few-shot

🎯 O que é calibrar

  • Sem referência, cada avaliação pode variar
  • Inclua no prompt 1–2 exemplos de resposta com a nota "certa"
  • A IA passa a seguir o mesmo padrão — isso é confiabilidade (um RNF)
  • Poucos exemplos, bem escolhidos, valem mais que muitos

🧪 Exemplo no prompt

Exemplo de nota 5: "resposta com dado concreto e resultado"
Exemplo de nota 2: "resposta genérica, sem evidência"

Agora avalie no mesmo padrão: [resposta]

Os exemplos ensinam a régua. Duas respostas parecidas passam a receber notas parecidas.

IA responsável no recrutamento

Num agente que avalia pessoas, isto não é opcional

Ética · RNF

⚠️ Três cuidados obrigatórios

  • Viés: instrua a IA a ignorar idade, gênero, origem, aparência — avaliar só o que se relaciona ao trabalho.
  • Privacidade (LGPD): não exponha dados pessoais do candidato; use apenas o necessário para a avaliação.
  • Decisão humana: a IA recomenda, o recrutador decide. Nunca uma rejeição automática.

🔗 Onde isso entra

  • São restrições do prompt — vêm direto dos RNF da spec (Dia 1)
  • Conectam com a ISO/IEC 25010: segurança e confiabilidade
  • Um bom prompt escreve isso de forma explícita, não deixa implícito
Restrições: avalie apenas competências ligadas à vaga; não use dados pessoais sensíveis; sinalize incerteza em vez de inventar.

Iterar com casos de teste

Um prompt não nasce pronto — ele é ajustado com evidência

Iteração

🔁 O ciclo

  1. Escreva a v1 com as 6 partes
  2. Teste em pelo menos 3 respostas de exemplo
  3. Compare o output com o critério de aceite da spec
  4. Localize qual parte do prompt causou o desvio
  5. Ajuste só essa parte e teste de novo

🧪 Escolha bons casos de teste

  • Uma resposta forte (deve pontuar alto)
  • Uma resposta fraca (deve pontuar baixo)
  • Uma resposta no limite ou ambígua
  • Se a IA erra o caso limite, o prompt precisa de mais critério ou exemplo

Parametrização e falhas comuns

Um prompt que serve a várias vagas — e o que costuma quebrar

Robustez

🎛️ Parametrização

  • Marque o que muda com colchetes: [vaga], [critérios], [resposta]
  • O mesmo prompt passa a servir a várias vagas
  • No Lovable, cada parâmetro vira um campo do formulário
  • Teste com parâmetros diferentes para garantir que continua válido

Nota inconsistenteSem critérios ou exemplosAdicione critérios e calibração (few-shot)
Sinais de viésSem restrição de atributosProíba idade/gênero/origem explicitamente
Formato quebradoFormato não especificadoExija JSON/tabela e dê um exemplo

🔨 Mãos à obra — escrevam e iterem o prompt

Sprint em grupo · cronômetro de 30 min · escrever, testar e ajustar

⏱️ Metodologia Ativa

🧭 Passos do sprint

  1. Escrevam a v1 do prompt com as 6 partes
  2. Peçam saída estruturada (tabela ou JSON)
  3. Testem em 3 respostas: forte, fraca e no limite
  4. Comparem com o critério de aceite e ajustem uma parte por vez

👥 Papéis no grupo

  • Quem escreve o prompt (mantém a versão única)
  • Quem testa com as 3 respostas
  • Quem registra o que mudou a cada iteração

✋ Chamem o professor quando

  • O output não bate com o critério após 2 ajustes
  • A IA mostrar qualquer sinal de viés
  • Não souberem qual parte do prompt corrigir

🏁 Critério de aceite

Pronto quando
As 3 respostas produzem saída no formato definido, com notas coerentes e sem uso de atributos protegidos. Se não, o ajuste é no prompt — não no critério.

Checklist — o prompt está robusto?

Respondam "sim" a tudo antes de considerar o prompt pronto

Autoavaliação
  • Tem um papel claro para a IA?
  • Traz o contexto (critérios e perfil da vaga)?
  • A tarefa é um verbo claro, sem ambiguidade?
  • Define o formato de saída (tabela/JSON)?
  • Tem restrições de viés e privacidade?
  • Foi calibrado com ao menos um exemplo?
  • Foi testado em 3 respostas (forte, fraca, limite)?
  • A saída ficou consistente entre os testes?
  • Está parametrizado para outras vagas?
  • Deixa claro que a decisão é humana?
