Dia 2 — APIs & Regra de Negócio

Conectar o agente a um serviço externo e dar-lhe uma lógica
Slides ← Voltar

🎯 Objetivo do dia

Conectar o agente a um serviço externo via API e dar-lhe uma lógica de negócio testável de ponta a ponta.

📚 Conteúdo

O que vamos cobrir

  1. Conceitos: endpoints, métodos (GET/POST), headers, auth, payload, status codes.
  2. Montagem de requisições e tratamento de respostas/erros (timeouts, retries simples, mensagens amigáveis).
  3. Criação de uma regra de negócio mensurável (validação, cálculo, elegibilidade, priorização, classificação, roteamento).
  4. Testes manuais e checklist de validação (cenários feliz + cenários de falha).

🏁 Entregável

Entrega do dia Integração com API (real ou mock) rodando, com pelo menos 1 regra de negócio implementada e testada de ponta a ponta, documentada com exemplos de entrada/saída.
