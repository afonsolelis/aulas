🎯 Goal of the day

This is the longest day of the camp: four development blocks. The goal is to have, by 4:00 PM, a functional MVP — user input → prompt → correct output — that meets the spec's acceptance criteria and is published via an externally accessible link.

🗓️ Blocks of the day

Time organization

  1. 09:30 – 11:00 — Instruction and scope alignment.
  2. 11:00 – 12:00 — 1st development block.
  3. 13:30 – 15:00 — 2nd development block.
  4. 15:10 – 16:00 — 3rd development block.

🔄 How to run the development

Use a simple kanban with three columns — To do, Doing, and Done — and the golden rule: at most one task in "Doing" per person. This keeps focus and makes progress visible to the whole group.

Checkpoints — goals per time block

  • By 12:00: input interface working; spec fields mapped as parameters.
  • By 15:00: prompt integrated and generating output; output appearing in the interface.
  • By 16:00: published via link and tested against the acceptance criteria.

When to call the professor

  • More than 30 minutes stuck on the same bug.
  • A long discussion with no one writing code.
  • Inconsistent output and the group cannot explain why.

✅ Checklist before considering it done

Verification
Can the user enter the input data?
Does the prompt receive that data as parameters?
Does the output appear legibly in the interface?
Does the output meet the spec's acceptance criteria?
Is the public link working?
