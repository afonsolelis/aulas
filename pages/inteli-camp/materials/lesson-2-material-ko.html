2일차 — 모델링에서 출발하는 프롬프트

스펙을 AI를 위한 정확한 지시로 전환하기
슬라이드 ← 돌아가기

🎯 오늘의 목표

1일차에 검증된 스펙을 구조화된 프롬프트로 변환하고, 출력물이 수용 기준을 충족할 때까지 테스트하는 것입니다. 프롬프트는 흩어진 텍스트가 아닙니다. AI가 이해하고 따르는 지시로 모델링을 직접 번역한 것입니다.

🔧 좋은 프롬프트의 해부

효과적인 프롬프트는 다섯 부분으로 이루어집니다. 각 부분은 여러분이 이미 스펙에서 정의한 요소에 대응합니다.

효과적인 프롬프트의 5가지 부분

  1. 역할(role): 이 맥락에서 AI가 누구인가 — 전문가, 어시스턴트, 검토자.
  2. 맥락: AI가 문제와 파트너에 대해 알아야 할 것.
  3. 과제: AI가 정확히 무엇을 해야 하는가 — 명확한 동사, 모호함 없이.
  4. 매개변수: 사용할 때마다 바뀌는 변수(사용자 입력, 설정).
  5. 제약: AI가 할 수 없는 것 — 스펙에서 직접 도출.

📋 스펙에서 프롬프트로

1일차의 예시 스펙을 고려하십시오(행위자: 마케팅 분석가, 입력: 브리핑, 출력: 카피 3가지 변형, 제약: 금지 단어 없이 최대 280자). 이것은 다음과 같이 번역됩니다:

당신은 [파트너 업종] 전문 카피라이터입니다.
브리핑을 바탕으로: [브리핑],
SNS용 카피 3가지 변형을 만드십시오.
규칙: 변형당 최대 280자, [금지 단어] 사용 금지,
어조 [어조] 유지.

대괄호 안의 부분이 매개변수입니다. 사용할 때마다 바뀌는 것입니다. Lovable에서는 각 매개변수가 사용자가 채우는 폼 필드가 됩니다.

🔁 프롬프트가 작동할 때까지 반복하는 법

프롬프트는 첫 버전에서 좋은 경우가 드뭅니다. 그래도 괜찮습니다. 효율적인 그룹을 구별짓는 것은 반복의 방법입니다.

반복 순환

  1. 작성하십시오. 다섯 부분으로 첫 버전을.
  2. 테스트하십시오. 서로 다른 최소 세 개의 입력으로.
  3. 비교하십시오. 출력물을 스펙의 수용 기준과.
  4. 식별하십시오. 프롬프트의 어느 부분이 이탈을 일으켰는지.
  5. 조정하십시오. 그 부분만 조정하고 다시 테스트하십시오.

가장 흔한 오류

  • 모호한 프롬프트: "제품에 대해 뭔가 써 줘" — AI가 지어냅니다.
  • 맥락 부재: AI가 파트너도 대상 고객도 모릅니다.
  • 제약 없음: AI가 어조와 형식의 한계를 무시합니다.
  • 한 번에 여러 부분 변경: 무엇이 효과가 있었는지 알 수 없게 만듭니다.
