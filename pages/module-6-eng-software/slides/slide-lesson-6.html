Por que mobile-first em recomendação 📱
Contexto de uso do espetáculo · sessão curta · em movimento · push notification

🧭 Contexto de uso favorece mobile

  • Em movimento: usuário decide a saída no metrô, na fila do café.
  • Sessão curta: 30s a 2min — interface tem que ser rápida.
  • Push notification: recomendação proativa (novo espetáculo, desconto).
  • Geolocalização: "espetáculos perto de mim hoje à noite".
  • Câmera + leitor de QR: ingresso digital na entrada.

⚠️ Limitações que mudam a arquitetura

  • Bateria: evitar polling agressivo, usar push.
  • Rede instável: cache offline obrigatório (TanStack Query, AsyncStorage).
  • Tela pequena: hierarquia forte, scroll vertical, top-K curto.
  • RAM/CPU limitados: FlatList virtualizada, imagens otimizadas.
  • Versão de SO fragmentada: testar em iOS + Android antigos.

React Native em 1 slide ⚛️
JS rodando em runtime nativo · bridge JSI · componentes nativos reais

🧠 O que é

Framework do Meta para construir apps mobile iOS e Android com JavaScript/TypeScript. Os componentes (<View>, <Text>) são traduzidos em componentes nativos (UIView, android.view.View) — não são WebView.

🌉 Arquitetura: bridge antiga → JSI

  • Antiga (bridge): JS ⇄ JSON serializado ⇄ nativo. Lento.
  • Nova (JSI · 2024+): JS chama C++ direto via JavaScript Interface. Sem serialização.
  • Hermes: motor JS otimizado para mobile (startup menor).
  • Fabric + Turbo Modules: renderização e módulos sob a JSI.

🆚 RN vs Flutter — diferença em 1 linha

RN usa os componentes nativos do SO (UIButton, Material). Flutter desenha tudo num canvas próprio (Skia) — visual 100% controlado, mas sem look-and-feel nativo automático.

App.tsx
import { View, Text } from 'react-native'; export default function App() { return ( <View style={{ padding: 16 }}> <Text>Aventura · Espetáculos</Text> </View> ); }
MVVM · O padrão arquitetural 🏛️
View ⇄ ViewModel → Model · binding bidirecional · View não conhece o Model
View

O que o usuário vê

Componente puro de UI. Não tem lógica de negócio. Renderiza estado e dispara eventos.

  • JSX / TSX
  • Não conhece o Model
  • Reage a mudanças do VM
ViewModel

Estado + ações da tela

Mantém o estado da View, chama o Model, expõe ações. Sem JSX. Em React: um custom hook.

  • useState / useReducer
  • Faz orquestração
  • Testável sem UI
Model

Dados + regras de domínio

Tipos, validação, fetch da API. Não conhece a View nem o ViewModel. Reusável em qualquer tela.

  • type / interface
  • fetch / axios
  • Camada de domínio

🆚 MVVM vs MVC

No MVC clássico a Controller orquestra tudo e a View conhece o Model. No MVVM a ViewModel isola — a View só sabe falar com o VM. Reduz acoplamento, ganha testabilidade.

🔁 Binding bidirecional

View → VM: input do usuário muda estado (ex.: digitou no campo). VM → View: estado novo re-renderiza a UI. Em React isso é o ciclo natural de useState + re-render.

Implementação · ViewModel (custom hook) 🪝
Mantém estado, chama o Model, expõe ações · sem JSX
viewmodels
useEspetaculos
ViewModel.ts
// viewmodels/useEspetaculosViewModel.ts import { useEffect, useState } from 'react'; import { Espetaculo, fetchEspetaculos } from '../model/Espetaculo'; export function useEspetaculosViewModel() { const [items, setItems] = useState<Espetaculo[]>([]); const [loading, setLoading] = useState(true); const [error, setError] = useState<string | null>(null); async function load() { try { setLoading(true); setError(null); setItems(await fetchEspetaculos()); } catch (e: any) { setError(e.message); } finally { setLoading(false); } } useEffect(() => { load(); }, []); return { items, loading, error, reload: load }; }
▶ jest test
terminal
PASS · useEspetaculosViewModel ✓ inicia com loading=true ✓ popula items quando fetch resolve ✓ seta error quando fetch rejeita ✓ reload reexecuta o fetch

💡 Por que custom hook? No React funcional, hooks são ViewModels. Carregam estado da tela, expõem ações, não tem JSX. Testáveis com @testing-library/react-hooks sem renderizar UI — exatamente o que MVVM promete.

Implementação · View (componente RN) 📱
Consome o hook · só renderiza · sem fetch, sem state local de domínio

// views/EspetaculosScreen.tsx import { FlatList, Text, View, ActivityIndicator } from 'react-native'; import { useEspetaculosViewModel } from '../viewmodels/useEspetaculosViewModel'; export default function EspetaculosScreen() { const { items, loading, error, reload } = useEspetaculosViewModel(); if (loading) return <ActivityIndicator size="large" />; if (error) return <Text>Erro: {error}</Text>; return ( <FlatList data={items} keyExtractor={(i) => i.id} onRefresh={reload} refreshing={loading} renderItem={({ item }) => ( <View style={{ padding: 12 }}> <Text>{item.nome} ★ {item.rating}</Text> </View> )} /> ); }

Estado, navegação e cache 🗂️
As 3 decisões que toda tela mobile resolve antes de virar produção

🧠 Estado · do local ao global

  • useState · estado local simples
  • useReducer · transições complexas
  • Context · estado de tela compartilhado
  • Zustand · global leve (sem boilerplate)
  • Redux Toolkit · time-travel, devtools

🧭 Navegação · React Navigation

  • Stack: push/pop entre telas
  • Tab: abas inferiores
  • Drawer: menu lateral
  • Deep linking aventura://espetaculo/42
  • Tipagem com RootStackParamList

💾 Cache + offline

  • AsyncStorage · key-value local
  • TanStack Query · cache + refetch + retry
  • SWR · stale-while-revalidate
  • NetInfo para detectar offline
  • Estratégia: stale-cache + revalidate

📌 Regra de bolso: comece com useState dentro do hook ViewModel. Promova para Zustand quando 2+ telas compartilharem o mesmo estado. Não use Redux só porque é famoso.

Screen.tsx
// tudo num arquivo só 😱 import { useEffect, useState } from 'react'; import { FlatList, Text, View, ActivityIndicator } from 'react-native'; export default function EspetaculosScreen() { const [items, setItems] = useState<any[]>([]); // estado → VM? const [loading, setLoading] = useState(true); useEffect(() => { // fetch → Model? fetch('https://api.aventura.app/espetaculos') .then(r => r.json()) .then(d => { setItems(d); setLoading(false); }); }, []); if (loading) return <ActivityIndicator size="large" />; return ( <FlatList data={items} keyExtractor={(i) => i.id} renderItem={({ item }) => <Text>{item.nome} ★ {item.rating}</Text>} /> ); }
