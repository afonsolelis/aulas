O Problema · Componente Acoplado 🍝
Tela de recomendações com fetch, estado e UI no mesmo arquivo · 100 linhas que ninguém quer testar
📄
app/screens/RecomendacoesScreen.tsx · 109 linhas · 0 testes · acoplado
export default function RecomendacoesScreen() { const [data, setData] = useState<Espetaculo[]>([]); const [loading, setLoading] = useState(true); const [error, setError] = useState<string | null>(null); useEffect(() => { // 🚨 fetch DENTRO do componente fetch('https://api.aventura.com/recs?u=123&k=10') .then(r => r.json()) .then(json => { setData(json); setLoading(false); }) .catch(e => { setError(e.message); setLoading(false); }); }, []); if (loading) return <ActivityIndicator />; // 🚨 lógica de UI if (error) return <Text>{error}</Text>; return <FlatList data={data} renderItem={... 60 linhas inline ...} />; }

🧪 Testar é difícil

Pra testar o fetch, precisa montar todo o React Native + jsdom. Custo alto, retorno baixo.

♻️ Reusar é impossível

Outra tela quer a mesma lista? Vai duplicar 100 linhas ou fazer copy-paste.

🐛 Debug é caos

Bug no estado? Bug no render? Bug na API? Tudo no mesmo arquivo.

'http://localhost:5000'; export async function fetchTopK(usuarioId: number, k = 10): Promise<Espetaculo[]> { const url = `${BASE_URL}/recommendations/${usuarioId}?k=${k}`; const res = await fetch(url); if (!res.ok) throw new Error(`Falha ao buscar recomendações (${res.status})`); return res.json() as Promise<Espetaculo[]>; }

📌 Princípio: a camada Model não importa nada de React Native. É TypeScript puro com fetch. Pode rodar em Node, em web, em mobile — em qualquer lugar que tenha fetch global.

Refatoração · Célula 2 · Camada ViewModel
viewmodels/useRecomendacoesViewModel.ts · custom hook que orquestra estado
In [2]:
viewmodels/useRecomendacoesViewModel.ts
import { useState, useEffect, useCallback } from 'react'; import { fetchTopK, Espetaculo } from '../services/RecomendacaoService'; export function useRecomendacoesViewModel(usuarioId: number) { const [data, setData] = useState<Espetaculo[]>([]); const [loading, setLoading] = useState(true); const [error, setError] = useState<string | null>(null); const recarregar = useCallback(async () => { setLoading(true); setError(null); try { setData(await fetchTopK(usuarioId)); } catch (e: any) { setError(e.message); } finally { setLoading(false); } }, [usuarioId]); useEffect(() => { recarregar(); }, [recarregar]); return { loading, data, error, recarregar }; }
▶ Type-checking
Out[2]:
✅ Hook expõe { loading, data, error, recarregar } — assinatura clara

🪝 Por que custom hook? O ViewModel é estado + ações. Em React Native, o jeito idiomático é hook. Não importa FlatList nem Text — só react e o serviço. É testável sem renderizar nada.
