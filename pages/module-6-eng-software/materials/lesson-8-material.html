Testes de Integração na API .NET
SlidesPlano← Voltar ao Módulo

Objetivo

Implementar testes de integração para um fluxo da API .NET do projeto, validando wiring entre camadas, contratos com o banco de dados e regras de negócio — entregando o resultado via Merge Request com rastreabilidade explícita a RF e RNF.

Pontos-chave

  • Pirâmide de testes: muitos unitários, alguns de integração, poucos E2E.
  • WebApplicationFactory<Program> sobe a aplicação em-process e permite trocar dependências (BD, e-mail, cache) na DI.
  • Cenário de sucesso valida o fluxo feliz (status 2xx + body esperado); cenário de erro valida violação de regra (4xx + ProblemDetails).
  • Banco em testes: SQLite InMemory para a maioria, Testcontainers quando o dialeto importa, Respawn para isolar entre testes.
  • CI/CD: dotnet test rodando em todo MR, com bloqueio de merge quando vermelho.
  • Rastreabilidade: a descrição do MR cita o RF (regra de negócio) e o RNF (eixo ISO 25010) que o teste protege.

Entrega da ponderada

Individualmente, cada aluno abre uma branch test/integration-{flow} no repositório do grupo, escreve um teste de sucesso + um teste de erro para um fluxo dos services, garante que o pipeline está verde, abre um Merge Request com a descrição preenchida (RF + RNF + rastreabilidade) e cola o link do MR na plataforma da disciplina.

Bibliografia

  • Integration tests in ASP.NET Core — Microsoft Learn.
  • xUnit · Getting Started — xunit.net.
  • Testcontainers for .NET — dotnet.testcontainers.org.
  • Jay Fields, Working Effectively with Unit Tests.
  • ISO/IEC 25010 — Quality model — referência rápida.
