Tópicos de BD: Stored Procedures e Functions
Slides← Voltar ao Módulo

Objetivo

Encapsular regras críticas de leitura e escrita dentro do banco de dados via stored procedures e functions, reduzindo round-trips, padronizando contratos entre serviços e simplificando o controle de transações.

Pontos-chave

  • Quando descer lógica para o BD (e quando NÃO descer).
  • Anatomia de uma CREATE PROCEDURE: parâmetros, defaults, OUTPUT, transação com TRY/CATCH.
  • Diferença entre function escalar (retorna 1 valor, usável em SELECT/WHERE) e function tabular (TVF — usável em FROM/JOIN).
  • Comparativo procedure × function: side-effects, retorno, uso em queries, transações.
  • Plano de execução: cache do plano, parameter sniffing, OPTION (RECOMPILE), importância dos índices que cobrem WHERE/ORDER BY.
  • Versionamento e CI/CD: migrações idempotentes (Flyway, Liquibase, EF Migrations), gates do GitLab CI, rollback.

Atividade prática (em sala · em grupo · não ponderada)

Criar o arquivo db/procedures.sql no repositório do grupo, com:

  1. 1 stored procedure que retorna top-N recomendações de um usuário (parâmetros: @userId, @N).
  2. 1 procedure que registra uma interação (INSERT em interacao + UPDATE em estatistica_item) dentro de uma transação implícita.
  3. 1 function escalar que calcula a média de ratings de um item.
  4. 1 function tabular que retorna os 5 itens mais parecidos com um item dado.
  5. README curto (db/README.md) explicando cada um, com exemplo de chamada (EXEC / SELECT).

Entrega: Merge Request feat(db): procedures e functions no repositório do grupo até o fim do dia.

⭐ Excelência: cobertura de erro (TRY/CATCH em SQL Server / EXCEPTION em PostgreSQL) + 1 índice criado para suportar a procedure principal.
