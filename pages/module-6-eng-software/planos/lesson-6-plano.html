Resumo

A aula 6 abre a frente de mobile. Apresentamos React Native como caminho para chegar ao nativo (iOS/Android) com a mesma base JS/TS, contrastando com app puramente nativo (Swift/Kotlin) e PWA. Em seguida estruturamos o app em MVVM: View apenas renderiza, ViewModel orquestra estado e regras de UI, Model conhece os serviços da API.

O fechamento aborda navegação (stack, tabs, drawer), gestão de estado (local, contexto, biblioteca dedicada) e o consumo dos serviços SOA via fetch/axios com tratamento de erro e loading.

Objetivos de aprendizagem

  • OA-1Diferenciar React Native, nativo puro e PWA, justificando trade-offs.
  • OA-2Estruturar o app em MVVM com responsabilidades claras por camada.
  • OA-3Implementar navegação com stack + tabs.
  • OA-4Conectar ViewModel à API SOA com loading e error states.
  • OA-5Aplicar componentes nativos (Touchable, FlatList) com performance em mente.
  • OA-6Avaliar escolhas de gestão de estado (local, Context, Zustand/Redux) para o porte do app.

Pré-requisitos

  • Node.js LTS instalado.
  • Expo CLI ou React Native CLI rodando o app exemplo no emulador/dispositivo.
  • API SOA com pelo menos 1 endpoint do recommendation service exposto.

Estratégias de metodologia ativa

  • Camadas como contrato — View nunca chama API direto; sempre passa pelo ViewModel.
  • Estado como decisão arquitetural — equipe defende em sala a escolha de gestão de estado.
  • Replicação contra API real — toda tela usa a API de fato, nunca mock estático.
Princípio orientador

Se a View tem fetch direto, o MVVM virou MV (sem o ponto chave que justifica o padrão).

Critério mínimo

Aluno que sair às 18h sem 2 telas no app consumindo a API real está em débito técnico para a aula 7 (TDD).

Conexão com a aula 7

A aula 7 traz Orientação a Objetos com TDD. O ViewModel passa a ser construído sob disciplina de Red-Green-Refactor.

  • O ViewModel ganha testes antes de novas funcionalidades.
  • Modelos de domínio (puro JS/TS) ficam mais isolados para serem testáveis.
Para o autoestudo da aula 7

Antes das 10h: ler material da aula 7, instalar Jest/Vitest e rodar 1 teste falho de propósito.
