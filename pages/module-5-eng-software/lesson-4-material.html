Security in Web and Cloud Systems

Lesson Goal

Recognize critical web and cloud security risks, strengthen authentication and authorization practices, and apply basic protection mechanisms for applications and infrastructure.

1) OWASP Top 10

The OWASP Top 10 is one of the most practical references for software teams because it concentrates the most recurrent and impactful web security risks. It includes broken access control, cryptographic failures, injection, insecure design, misconfiguration, vulnerable components, authentication failures, integrity failures, poor logging, and SSRF.

2) Authentication and Authorization

Authentication answers who the user is. Authorization answers what the authenticated user is allowed to do. These are different concerns and both must be treated explicitly.

  • Prefer strong password hashing with bcrypt or argon2.
  • Use MFA for privileged or sensitive accounts.
  • Validate access on the server side for every protected request.

3) Cloud Security

Cloud environments follow a shared responsibility model. The provider secures the physical and foundational layers. The customer remains responsible for identity, data, application security, and configuration hygiene.

  • Use least-privilege IAM policies.
  • Store secrets in dedicated secret management solutions.
  • Keep databases in private networks when possible.
  • Audit logs should always be enabled.

4) Encryption

Data should be protected in transit with HTTPS/TLS and protected at rest with encrypted storage. Passwords are a special case: they should be hashed, not encrypted reversibly.
