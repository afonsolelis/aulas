Objetivo da Aula

Identificar as principais vulnerabilidades de aplicações web, aplicar boas práticas de autenticação/autorização e compreender o modelo de segurança compartilhada em nuvem.

1) OWASP Top 10

A OWASP (Open Web Application Security Project) lista as 10 vulnerabilidades mais críticas em aplicações web. Todo desenvolvedor precisa conhecê-las.

  1. A01 — Broken Access Control: usuários acessam recursos além de seu nível de permissão.
  2. A02 — Cryptographic Failures: dados sensíveis transmitidos sem criptografia ou com algoritmos fracos.
  3. A03 — Injection: SQL, NoSQL, OS injection — entrada do usuário executada como comando.
  4. A04 — Insecure Design: ausência de modelagem de ameaças desde a concepção.
  5. A05 — Security Misconfiguration: permissões padrão, headers ausentes, erros expostos.
  6. A06 — Vulnerable Components: dependências desatualizadas com CVEs conhecidos.
  7. A07 — Auth Failures: senhas fracas, sessões sem expiração, brute force sem proteção.
  8. A08 — Software & Data Integrity: pipelines CI/CD sem verificação de integridade.
  9. A09 — Logging Failures: ataques não detectados por ausência de logs.
  10. A10 — SSRF: servidor faz requisições a recursos internos manipulados pelo usuário.

2) Autenticação e Autorização

Autenticação: quem é você?  |  Autorização: o que você pode fazer?

JWT — JSON Web Token
# Gerar token (Python)
import jwt, datetime

payload = {
    "sub": "user_123",
    "role": "admin",
    "exp": datetime.datetime.utcnow() + datetime.timedelta(hours=1)
}
token = jwt.encode(payload, SECRET_KEY, algorithm="HS256")

# Verificar token
decoded = jwt.decode(token, SECRET_KEY, algorithms=["HS256"])

Boas práticas:

  • Use bcrypt ou argon2 para hashing de senhas — nunca MD5/SHA1.
  • Implemente MFA para contas privilegiadas.
  • Tokens JWT com expiração curta (15–60 min) + refresh token.
  • Valide permissões no backend em toda requisição.

3) Segurança em Nuvem

Modelo de Responsabilidade Compartilhada: o provedor cuida da infraestrutura física; o cliente cuida dos dados, identidade e aplicação.

  • Use IAM com permissões mínimas (least privilege).
  • Armazene segredos no Secrets Manager — nunca em variáveis de ambiente hardcoded no código.
  • Habilite logs de auditoria (CloudTrail, GCP Audit Logs).
  • Coloque bancos de dados em sub-redes privadas (sem IP público).
  • Security Groups — abra apenas as portas necessárias.

4) HTTPS e Criptografia

  • Use TLS 1.3 — versões anteriores têm vulnerabilidades conhecidas.
  • Certificados gratuitos com Let's Encrypt + renovação automática (Certbot).
  • HSTS — força HTTPS no browser por um período definido.
  • Criptografia em repouso: AES-256 para volumes de disco e backups.

5) Checklist de Segurança Mínima

  • HTTPS em todos os endpoints
  • Certificado válido e renovado
  • Senhas hasheadas com bcrypt/argon2
  • Nenhum segredo no código-fonte ou git
  • Dependências verificadas com pip audit ou npm audit
  • Logs de acesso e erros habilitados
  • Validação de entrada em todo dado externo

6) Entregável da Aula 3

  • Implementar autenticação com JWT em uma API simples (FastAPI ou Express).
  • Adicionar uma rota protegida que valide o token antes de responder.
  • Executar pip audit (Python) ou npm audit (Node) e corrigir ao menos uma vulnerabilidade encontrada.
