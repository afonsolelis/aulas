Fundamentos de Sistema Operacional

⚙️ Responsabilidades do SO

  • Gerenciamento de processos — scheduling da CPU
  • Gerenciamento de memória — RAM e memória virtual
  • Sistema de arquivos — abstração de armazenamento
  • Gerenciamento de I/O — dispositivos e drivers
  • Segurança — permissões e isolamento

🔄 Processo vs Thread

Processo: unidade de execução independente com seu próprio espaço de memória.


Thread: unidade leve dentro de um processo, compartilha memória, mais rápida para comunicação, mas introduz condições de corrida e sincronização.

Docker: VM vs Container

🖥️ Máquinas Virtuais

  • Emulam hardware completo
  • Cada VM tem seu próprio SO
  • Isolamento total, maior segurança
  • Pesadas: GB de memória
  • Boot em minutos

🐋 Containers Docker

  • Compartilham o kernel do host
  • Isolamento por namespaces e cgroups
  • Levíssimos: MB de memória
  • Boot em segundos
  • Portabilidade: "works on my machine" resolvido
# Puxar imagem e rodar container docker pull node:20-alpine docker run -d -p 3000:3000 --name minha-api node:20-alpine docker ps # listar containers em execução docker logs minha-api

Dockerfile Multi-stage Build

# Stage 1: Build FROM node:20-alpine AS builder WORKDIR /app COPY package*.json ./ RUN npm ci COPY . . RUN npm run build # Stage 2: Produção (imagem final menor) FROM node:20-alpine AS production WORKDIR /app ENV NODE_ENV=production COPY --from=builder /app/dist ./dist COPY --from=builder /app/node_modules ./node_modules EXPOSE 3000 CMD ["node", "dist/index.js"]

✅ Vantagens

  • Imagem de produção sem ferramentas de build
  • Redução drástica de tamanho
  • Menor superfície de ataque
  • Separação clara dev/prod
