JWT — JSON Web Token

🏗️ Estrutura do JWT

  • Header: algoritmo de assinatura (HS256, RS256)
  • Payload: claims (sub, iat, exp, roles)
  • Signature: hash do header + payload com secret

⚠️ Boas Práticas

  • Expiration curta (15-60 min)
  • Refresh token em storage seguro
  • Nunca armazenar secrets no payload
import jwt from 'jsonwebtoken'; // Gerar token const token = jwt.sign( { sub: user.id, role: user.role }, process.env.JWT_SECRET!, { expiresIn: '15m' } ); // Verificar token (middleware) export function authMiddleware(req, res, next) { const token = req.headers.authorization?.split(' ')[1]; if (!token) return res.status(401).json({ error: 'Unauthorized' }); try { req.user = jwt.verify(token, process.env.JWT_SECRET!); next(); } catch { res.status(401).json({ error: 'Invalid token' }); } }

Segurança em Containers e Cloud

🐋 Boas Práticas Docker

  • Use imagens oficiais e minimais (alpine)
  • Execute como usuário não-root
  • Faça scanning de vulnerabilidades (docker scout)
  • Nunca copie secrets para a imagem
  • Use multi-stage build para reduzir superfície

🔑 Gerenciamento de Secrets

  • Variáveis de ambiente via .env (nunca commitado)
  • AWS Secrets Manager, HashiCorp Vault
  • Docker Secrets para stacks Swarm
  • GitHub Secrets para CI/CD
Regra de Ouro: Trate secrets como senhas. Nunca em código, nunca em logs, nunca em imagens Docker. Rotacione regularmente.
