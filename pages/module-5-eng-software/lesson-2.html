OS Fundamentals
The software layer between hardware and applications

Core Responsibilities

  • Process management — scheduling CPU time
  • Memory management — RAM allocation, virtual memory
  • File system — storage abstraction
  • I/O management — devices and drivers
  • Security — user permissions, isolation

Process vs Thread

Process: independent execution unit with its own memory space.

Thread: lightweight unit within a process, shares memory — faster communication, but race conditions.
VMs vs Containers
Two approaches to isolation and portability

Virtual Machines

  • Full OS copy per VM
  • Strong isolation (hypervisor)
  • Heavy: GBs of disk, minutes to boot
  • Tools: VMware, VirtualBox, Hyper-V

Containers

  • Share host OS kernel
  • Process-level isolation (namespaces + cgroups)
  • Lightweight: MBs, seconds to start
  • Tools: Docker, Podman, containerd
Containers are not "better" than VMs — they serve different purposes. Production systems often combine both.
Docker in Practice
Build, Ship, Run — anywhere

Core Concepts

  • Image — immutable template (Dockerfile)
  • Container — running instance of an image
  • Registry — Docker Hub, GHCR, ECR
  • Volume — persistent data outside container
  • Network — container communication

Essential Commands

# Build image docker build -t myapp:1.0 . # Run container docker run -p 8080:80 myapp:1.0 # List running containers docker ps # Stop container docker stop <id> # View logs docker logs <id>
Writing a Dockerfile
Defining your application environment as code

Python Example

FROM python:3.12-slim WORKDIR /app COPY requirements.txt . RUN pip install -r requirements.txt COPY . . EXPOSE 8000 CMD ["python", "main.py"]

Best Practices

  • Use slim or alpine base images
  • Copy dependencies before source code (layer cache)
  • One process per container
  • Use .dockerignore to exclude files
  • Never store secrets in the image
