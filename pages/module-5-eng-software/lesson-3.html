Authentication vs Authorization
Who are you? What can you do?

Authentication

  • Password + hashing (bcrypt, argon2)
  • Multi-Factor Authentication (MFA)
  • OAuth 2.0 / OpenID Connect
  • JWT — stateless token-based auth
# JWT structure header.payload.signature # Verify in Python import jwt payload = jwt.decode( token, SECRET, algorithms=["HS256"] )

Authorization

  • RBAC — Role-Based Access Control
  • ABAC — Attribute-Based Access Control
  • Least privilege principle
  • Always validate on the server
Never trust the client. A user can modify browser-side tokens — always re-validate on every request.
Cloud Security Best Practices
Shared responsibility model and practical defenses

Shared Responsibility

  • Provider: physical infra, hypervisor, network
  • Customer: OS, data, identity, application

Key Practices

  • IAM — minimal permissions per service
  • Secrets Manager — never env vars in code
  • Enable logging (CloudTrail, Stackdriver)
  • Use VPC / private subnets for databases

Common Mistakes

  • Public S3 buckets with sensitive data
  • Wide-open security groups (0.0.0.0/0)
  • Hardcoded credentials in git repos
  • No encryption at rest or in transit
  • Missing audit logs
Scan your repositories for secrets: git-secrets, truffleHog, GitHub Secret Scanning.
Encryption and HTTPS
Protecting data in transit and at rest

TLS / HTTPS

  • TLS 1.3 — current standard
  • Certificate: validates server identity
  • Let's Encrypt — free, automated certs
  • HSTS — force HTTPS in browser

At Rest

  • AES-256 for stored data
  • Encrypt database volumes
  • Hash passwords (never encrypt)

Practical Checklist

✅ HTTPS everywhere
✅ Valid, renewed certificates
✅ Passwords hashed with bcrypt/argon2
✅ Sensitive data encrypted at rest
✅ No secrets in source code
✅ Input validation and sanitization
✅ Dependency vulnerability scanning
