What are Data Structures?
Organization and storage of data for efficient access and modification

Linear Structures

  • Array — contiguous memory, O(1) access by index
  • Linked List — nodes with pointers, O(1) insert/delete at head
  • Stack (LIFO) — push/pop, used in recursion and undo
  • Queue (FIFO) — enqueue/dequeue, used in scheduling

Why does it matter?

Choosing the wrong data structure can make an O(1) operation become O(n²). Every software engineering decision starts here.
  • Memory efficiency
  • Algorithm performance
  • Code readability and maintainability
Trees and Graphs
Non-linear structures for hierarchical and relational data

Trees

  • Binary Tree — max 2 children per node
  • BST — left < root < right, O(log n) search
  • Heap — priority queue, used in sorting
  • Trie — prefix-based, used in autocomplete

Graphs

  • Directed / Undirected
  • Weighted / Unweighted
  • BFS — shortest path, level order
  • DFS — cycle detection, topological sort
Real-world use: social networks (graphs), file systems (trees), databases (B-trees), DNS (trie).
Complexity Analysis — Big O
Measuring algorithm efficiency as input grows

Common Complexities

  • O(1) — constant: array index access
  • O(log n) — logarithmic: binary search
  • O(n) — linear: list traversal
  • O(n log n) — merge sort, quicksort avg
  • O(n²) — bubble sort, nested loops

Quick Reference

Structure | Access | Search | Insert ------------- | ------ | ------ | ------ Array | O(1) | O(n) | O(n) Linked List | O(n) | O(n) | O(1) Hash Table | O(1) | O(1) | O(1) BST (avg) | O(log) | O(log) | O(log) Stack/Queue | O(n) | O(n) | O(1)
Practical Implementation
Stack and Queue in Python

Stack (LIFO)

stack = [] # Push stack.append("a") stack.append("b") stack.append("c") # Pop top = stack.pop() # "c" # Peek peek = stack[-1] # "b"

Queue (FIFO)

from collections import deque queue = deque() # Enqueue queue.append("task1") queue.append("task2") # Dequeue first = queue.popleft() # "task1" # Size size = len(queue) # 1
