Authentication vs Authorization
Who are you, and what are you allowed to do?

Authentication

  • Passwords with strong hashing
  • Multi-factor authentication
  • OAuth 2.0 and OpenID Connect
  • JWT-based identity propagation

Authorization

  • Role-based access control
  • Attribute-based rules
  • Least privilege by default
  • Server-side validation on every request
Never trust the client alone. Permissions must always be validated on the server.
Cloud Security Best Practices
Security in the cloud follows a shared responsibility model

Customer Responsibilities

  • Identity and access management
  • Application and data protection
  • Secrets handling
  • Security group and network rules

Common Mistakes

  • Public data storage buckets
  • Hardcoded credentials in repositories
  • Wide-open inbound rules
  • Missing audit trails
Security in the cloud is not delegated completely to the provider. Misconfiguration is one of the most common causes of exposure.
Encryption and HTTPS
Protecting data in transit and at rest

In Transit

  • TLS 1.3 and modern HTTPS defaults
  • Certificate validation
  • HSTS and secure headers

At Rest

  • Encrypted disks and volumes
  • Protected backups
  • Passwords hashed, not encrypted
