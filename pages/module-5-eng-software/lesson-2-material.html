Objetivo da Aula

Compreender os fundamentos de sistemas operacionais, diferenciar VMs de containers, e criar e executar um container Docker com uma aplicação simples.

1) Fundamentos de Sistemas Operacionais

O sistema operacional é a camada de software que gerencia os recursos de hardware e fornece serviços para os programas.

  • Processos: unidade de execução independente com espaço de memória próprio.
  • Threads: unidade leve dentro de um processo, compartilha memória — mais rápido, mas exige cuidado com condições de corrida.
  • Gerenciamento de memória: RAM, memória virtual, paginação.
  • Escalonamento: Round Robin, Priority Scheduling, CFS (Linux).
# Verificar processos no Linux
ps aux

# Verificar uso de memória
free -h

# Verificar uso de CPU
top

3) Docker — Conceitos e Comandos

  • Image: template imutável definido pelo Dockerfile.
  • Container: instância em execução de uma imagem.
  • Volume: dados persistentes fora do container.
  • Network: comunicação entre containers.
# Construir imagem
docker build -t minha-app:1.0 .

# Executar container
docker run -d -p 8080:80 minha-app:1.0

# Listar containers ativos
docker ps

# Ver logs
docker logs <container_id>

# Parar e remover
docker stop <id>
docker rm <id>

4) Escrevendo um Dockerfile

FROM python:3.12-slim

WORKDIR /app

# Copiar dependências antes do código (cache de camadas)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copiar código-fonte
COPY . .

EXPOSE 8000

CMD ["python", "main.py"]

Boas práticas: use imagens slim/alpine, aproveite o cache de camadas, nunca armazene segredos na imagem, use .dockerignore.
