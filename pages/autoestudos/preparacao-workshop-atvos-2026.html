Preparação para o Workshop ATVOS (2026)

Guia estruturado de direcionamentos e perguntas para os alunos se prepararem de forma eficiente para o workshop do projeto da Atvos.

Para preparar os alunos de forma eficiente para o workshop do projeto da Atvos, estruturei os direcionamentos e perguntas com base nas informações do escopo, requisitos técnicos, modelagem de dados e regras de negócio exigidas.

Aqui estão os tópicos essenciais que eles devem dominar.

📋 Antes do workshop: brainstorming coletivo no Google Sheets

Para que o workshop com a Atvos seja o mais produtivo possível e para evitar que grupos diferentes façam a mesma pergunta, cada turma deve organizar seu preparo em um Google Sheets compartilhado.

  1. Criem um Google Sheets compartilhado entre todos os grupos da turma (com permissão de edição para a turma toda).
  2. Leiam o TAPI (Termo de Abertura do Projeto Integrador) com atenção — ele é a base para o brainstorming e para mapear as lacunas de informação que vocês ainda têm.
  3. Façam o brainstorming em grupo, listando hipóteses de solução, dúvidas técnicas, dúvidas de negócio e suposições que precisam ser validadas com o parceiro.
  4. Toda pergunta que vocês pretendem fazer ao cliente no workshop deve ser registrada no Sheets, identificando o grupo autor, o tópico (negócio, escopo, dados, técnico, ODS, etc.) e a pergunta em si.
  5. Antes de adicionar uma pergunta nova, confiram as perguntas já cadastradas pelos outros grupos. Se a pergunta já existe, apoiem a do outro grupo (upvote, comentário de complemento) em vez de duplicar.
Por que isso importa? O tempo do cliente no workshop é limitado. Perguntas duplicadas desperdiçam esse tempo e tiram espaço de dúvidas únicas e estratégicas de cada grupo. Centralizar no Sheets garante cobertura ampla, evita sobreposição e distribui melhor o tempo de fala entre os times.

1. Entendimento do Negócio e do Problema

Os alunos precisam compreender a dor do cliente e o perfil da empresa para propor uma solução adequada.

Direcionamento: Estudar o cenário da Atvos, que sofre com a falta de governança de dados, onde as informações dos parceiros estão dispersas em diferentes controles manuais (planilhas), dificultando a visualização e a tomada de decisão. O público-alvo da aplicação não são os parceiros, mas sim os times internos (como a área de Negócios Agrícolas).
Perguntas de preparo
  1. Qual é a diferença fundamental no domínio de negócios entre um "Parceiro" e um "FoCa" (Fornecedor de Cana)?
  2. Quais são os principais prejuízos que os controles manuais atuais trazem para o histórico de relacionamento de longo prazo com os stakeholders?

2. Escopo do MVP e Restrições do Sistema

É crucial que os alunos alinhem as expectativas sobre o que será e o que não será desenvolvido.

Direcionamento: O foco do MVP (Produto Mínimo Viável) é construir um protótipo web funcional e enxuto para centralização, cadastro e visualização de dados e do histórico de interações, sem automações complexas. Eles devem ter total clareza sobre as restrições técnicas acordadas.
Perguntas de preparo
  1. Sabendo que o projeto não contemplará sistemas de autenticação (sign in / log in) e nem integração com WebAPIs externas, como vocês planejam estruturar a usabilidade para os times internos?
  2. Quais são as principais entregas esperadas para o Nível 1 (Fundacional) até o Nível 4 (Avançado)?

3. Modelagem de Dados e Saneamento das Planilhas

Como o projeto parte de planilhas desestruturadas, a governança dos dados é um desafio técnico central.

Direcionamento: O grupo deve analisar as observações de qualidade dos dados originais (ex: múltiplas fazendas cadastradas na mesma célula separadas por ";", presença de múltiplos CNPJs para o mesmo parceiro e inconsistências de grafia). A modelagem relacional será essencial para normalizar isso.
Perguntas de preparo
  1. Como vocês farão a normalização de campos multivalorados, como os CNPJs, contatos e fazendas, na transição das planilhas para o diagrama de Entidade-Relacionamento (ER)?
  2. No tratamento dos dados legados, como será o processo de deduplicação de parceiros com grafias diferentes, utilizando chaves como CPF ou CNPJ?

4. Requisitos Técnicos e Entregas das Aulas

O projeto está fortemente atrelado às entregas de um módulo acadêmico de Desenvolvimento Web (Back-end, Front-end, Banco de Dados, Testes e Redes).

Direcionamento: A solução deve seguir as boas práticas ensinadas nas aulas: arquitetura MVC, API RESTful com Express.js, Front-end consumindo dados assíncronos (Fetch API) gerenciando estados de UI (loading, sucesso, erro), e banco de dados relacional.
Perguntas de preparo
  1. Como vocês estruturarão os endpoints da API para garantir um tempo de resposta inferior a 200ms e tratamento adequado de erros com status codes?
  2. Para construir os Dashboards do Nível 4 (Estratégico/Avançado), como vocês usarão JOINs, GROUP BY e funções de agregação no SQL para extrair estatísticas (ex: evolução da base de parceiros, área contratada vs. potencial)?
  3. Como o plano de automação de testes com Jest ou Playwright garantirá uma cobertura mínima de 70% rodando em menos de 30 segundos?

5. Impacto Operacional e ODS (Agenda 2030)

O projeto será avaliado também pelo seu impacto prático e sustentável.

Direcionamento: Os alunos terão que preencher um checklist justificando como o projeto atende às necessidades práticas da Atvos (Impacto Operacional) e a quais das 5 dimensões dos Objetivos de Desenvolvimento Sustentável (ODS) da ONU ele se alinha: Pessoas, Planeta, Prosperidade, Paz e Parcerias.
Perguntas de preparo
  1. Como a centralização tecnológica dos parceiros da Atvos (empresa de bioenergia) se conecta indiretamente à dimensão "Planeta" ou à dimensão "Prosperidade" dos ODS?
