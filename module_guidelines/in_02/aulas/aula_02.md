# Aula 2 - BANCO DE DADOS I - CONCEITOS, MODELOS E SQL BÁSICO

## Descrição da Aula
Introdução à modelagem de dados e ao raciocínio relacional. A aula conecta o domínio do problema às entidades, atributos, relacionamentos e às primeiras consultas SQL.

---
**Visão RM-ODP:** Information  
**RF:** Mapear quais funcionalidades dependem de persistência de dados  
**RNF (8 eixos):** Confiabilidade + Segurança dos dados  
**Artefato:** Minimundo + ER/DER inicial + primeiros comandos SQL

### Onde queremos chegar
- [ ] Identificar entidades, atributos e relacionamentos do domínio
- [ ] Diferenciar modelo conceitual, lógico e físico
- [ ] Definir chaves primárias e estrangeiras
- [ ] Escrever consultas SQL básicas com `SELECT`, `WHERE`, `ORDER BY`
- [ ] Relacionar regras de negócio com restrições de dados

---
## Auto Estudo Recomendado (Tempo Estimado Total: 120 min)

1. **Modelagem entidade-relacionamento** (Estimativa: 40 min):
   - Entidades, atributos, cardinalidade e relacionamentos.
   **Por que estudar isso:** É a base para representar o domínio antes de criar tabelas.

2. **Modelo relacional** (Estimativa: 30 min):
   - Tabelas, tuplas, chaves e integridade referencial.
   **Por que estudar isso:** Permite traduzir o modelo conceitual para estruturas persistentes.

3. **SQL básico** (Estimativa: 30 min):
   - `SELECT`, `FROM`, `WHERE`, `ORDER BY`, `LIMIT`.
   **Por que estudar isso:** Introduz a leitura de dados logo no início da trilha de banco.

4. **Exercícios curtos de consulta** (Estimativa: 20 min):
   - Montar consultas simples com filtros e ordenação.
   **Por que estudar isso:** Consolida a passagem do diagrama para a manipulação prática dos dados.
