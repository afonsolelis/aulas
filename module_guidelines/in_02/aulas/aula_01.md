# Aula 1 - INTRODUÇÃO AOS SISTEMAS WEB

## Descrição da Aula
Visão geral de como um sistema web funciona, quais são suas camadas principais, como navegador e servidor se comunicam e como a disciplina será organizada ao longo da trilha.

---
**Visão RM-ODP:** Enterprise + Technology  
**RF:** Identificar o problema, os usuários e o fluxo principal do sistema  
**RNF (8 eixos):** Usabilidade + Desempenho + Segurança em nível introdutório  
**Artefato:** Mapa inicial do sistema web + lista preliminar de RF/RNF

### Onde queremos chegar
- [ ] Entender a diferença entre cliente, servidor, banco e rede
- [ ] Identificar usuários, objetivos e fluxo principal do sistema
- [ ] Relacionar páginas, ações do usuário e respostas esperadas
- [ ] Levantar RF e RNF iniciais do projeto
- [ ] Registrar um vocabulário básico do domínio

---
## Auto Estudo Recomendado (Tempo Estimado Total: 120 min)

1. **Conceitos de aplicações web** (Estimativa: 30 min):
   - Arquitetura cliente-servidor, requisição/resposta, HTTP, navegador e servidor.
   **Por que estudar isso:** Dá a base para compreender como front-end, back-end, banco e rede se conectam no projeto.

2. **Introdução a requisitos de software** (Estimativa: 30 min):
   - RF, RN e RNF; diferença entre necessidade do usuário e solução técnica.
   **Por que estudar isso:** Ajuda a começar a disciplina com foco em problema, valor e qualidade.

3. **Leitura guiada sobre sistemas web modernos** (Estimativa: 30 min):
   - Conceitos de URL, rota, formulário, API e persistência.
   **Por que estudar isso:** Prepara o terreno para as aulas seguintes de banco, back-end e front-end.

4. **Exploração prática do navegador** (Estimativa: 30 min):
   - Inspecionar páginas, elementos HTML e requisições na aba de rede.
   **Por que estudar isso:** Aproxima a teoria do funcionamento real de uma aplicação web.
